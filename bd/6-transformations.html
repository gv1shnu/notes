6. Transformations.

A transformation takes an RDD and returns a new one. They are lazy — nothing actually runs until an action asks for a result. Spark just records the recipe.

Transformation Effect
map(f) one element → one element
filter(f) keep elements where f is true
flatMap(f) one element → many (then flattened)
In code 
Python

words = sc.parallelize(["scala", "spark", "pyspark"])

words.map(lambda w: (w, 1))             # ('scala',1), ('spark',1), ...
words.filter(lambda w: "spark" in w)    # 'spark', 'pyspark'
lines.flatMap(lambda line: line.split(" "))   # each line → its words

None of these have run yet. They just build the lineage.
