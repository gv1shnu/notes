7. Actions.

An action is what actually triggers the work. It runs the whole lineage and returns a result to the driver.

Action Returns
show(n) first n rows of a DataFrame, as a table
collect() every element, as a list (careful: pulls it all to the driver)
count() number of elements
take(n) the first n elements
In code 
Python

df.show(5)        # first 5 rows
rdd.collect()     # [whole RDD as a list]
rdd.count()       # 8
rdd.take(3)       # first 3 elements

The pattern: build with transformations (lazy), then fire one action to make it run.
