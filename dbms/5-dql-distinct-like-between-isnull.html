Note: NULL means the value is absent — not zero, not an empty string, but genuinely unknown or missing. For example, a player who is unranked has no level to record. Storing 0 would be wrong (0 might mean something else). Storing an empty string would be imprecise. NULL is the correct representation of "no value."

DISTINCT

DISTINCT removes duplicate values. Without DISTINCT, querying SELECT level FROM players returns one row per player — you'd see "Radiant" twice and "Immortal" twice. DISTINCT collapses duplicates so you get each value once. 

MySQL / PostgreSQL

SELECT DISTINCT level
FROM players;

LIKE

LIKE performs pattern matching.

% means any number of characters.

In MySQL, LIKE is case-insensitive by default, so LIKE 'a%' matches both 'a' and 'A' prefixes. However, in PostgreSQL, quoted patterns are case-sensitive, so LIKE 'a%' matches only lowercase 'a' prefixes unless you use a case-insensitive operator such as ILIKE. 

MySQL / PostgreSQL

SELECT username
FROM players
WHERE username LIKE 'a%';

IN

IN checks multiple possible values. 

MySQL / PostgreSQL

SELECT username, region
FROM players
WHERE region = 'NA' OR region = 'BR' OR region = 'EU';

We can write a much cleaner code using the IN operator.

MySQL / PostgreSQL

SELECT username, region
FROM players
WHERE region IN ('NA', 'BR', 'EU');

BETWEEN

BETWEEN filters values inside a range. 

MySQL / PostgreSQL

SELECT username, score
FROM players
WHERE score >= 600 AND score <= 850;

Similar to what we did above, we can write a much readable code using the BETWEEN operator. 

MySQL / PostgreSQL

SELECT username, score
FROM players
WHERE score BETWEEN 600 AND 850;

IS NULL

IS NULL checks missing values. 

MySQL / PostgreSQL

SELECT username, score
FROM players
WHERE score = NULL;

NULL = NULL is not TRUE in SQL — it's NULL. NULL is not equal to anything, not even itself. So = NULL will never match any row. You must use IS NULL.

MySQL / PostgreSQL

SELECT username, score
FROM players
WHERE score IS NULL;
