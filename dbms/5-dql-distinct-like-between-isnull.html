Note: NULL means the value is absent — not zero, not an empty string, but genuinely unknown or missing. For example, a player who is unranked has no rank to record. Storing 0 would be wrong (0 might mean something else). Storing an empty string would be imprecise. NULL is the correct representation of "no value."

DISTINCT

DISTINCT removes duplicate values. Without DISTINCT, querying SELECT rank FROM players returns one row per player — you'd see "Radiant" twice and "Immortal" twice. DISTINCT collapses duplicates so you get each value once.

MySQL / PostgreSQL


SELECT DISTINCT rank
FROM players;

Output:

rank
Radiant
Immortal
NULL

LIKE

LIKE performs pattern matching.

% means any number of characters. 


SELECT username
FROM players
WHERE username LIKE 'a%';

Output:

username
aspas

IN

IN checks multiple possible values. 


SELECT username, region
FROM players
WHERE region = 'NA' OR region = 'BR' OR region = 'EU';

We can write a much cleaner code using the IN operator.


SELECT username, region
FROM players
WHERE region IN ('NA', 'BR', 'EU');

Output:

username region
TenZ NA
aspas BR
yay NA
leaf NA

BETWEEN

BETWEEN filters values inside a range. 


SELECT username, rr
FROM players
WHERE rr >= 600 AND rr <= 850;

Similar to what we did above, we can write a much readable code using the BETWEEN operator. 


SELECT username, rr
FROM players
WHERE rr BETWEEN 600 AND 850;

Output:

username rr
TenZ 842
aspas 811
yay 620

IS NULL

IS NULL checks missing values. 


SELECT username, rr
FROM players
WHERE rr = NULL;

NULL = NULL is not TRUE in SQL — it's NULL. NULL is not equal to anything, not even itself. So = NULL will never match any row. You must use IS NULL.


SELECT username, rr
FROM players
WHERE rr IS NULL;

Output:

username rr
leaf NULL

Summary

Command Purpose
DISTINCT Remove duplicates
LIKE Pattern matching
IN Multiple matching values
BETWEEN Range filtering
IS NULL Find missing values
