Installing MySQL on macOS

In this guide, MySQL Server is installed using Homebrew and accessed through both the terminal and MySQL Workbench.

Step 1: Verify Homebrew Installation

Check whether Homebrew is installed: 

brew --version

If Homebrew is not installed: 

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Step 2: Install MySQL Server

Install MySQL using Homebrew: 

brew install mysql

Verify the installation: 

mysql --version

Example output: 

mysql  Ver 9.x for macos...

Step 3: Start MySQL Service

Start MySQL as a background service: 

brew services start mysql

Check the service status: 

brew services list

Expected output: 

mysql started

Step 4: Connect Using the Terminal

Connect to the MySQL server: 

mysql -u root -p

Enter the root password when prompted.

If the connection succeeds, the prompt changes to: 

mysql>

Step 5: View Existing Databases

SHOW DATABASES;

Step 6: Create a New Database

CREATE DATABASE company;

Step 7: Use the Database

USE company;

Verify the selected database: 

SELECT DATABASE();

Step 8: View Existing Users

SELECT User, Host
FROM mysql.user;

Step 9: Exit MySQL

EXIT;

Using MySQL Workbench

Step 1: Install MySQL Workbench

Download and install MySQL Workbench from the official MySQL website.

Step 2: Create a New Connection

Open MySQL Workbench and click the + icon beside "MySQL Connections".

Use the following settings:

  • Connection Name: Local MySQL
  • Hostname: localhost
  • Port: 3306
  • Username: root

Click Test Connection and enter the password when prompted.

Step 3: Connect to the Server

Double-click the saved connection.

The SQL Editor will open and you can execute queries directly from the interface.

Step 4: Create a Database

CREATE DATABASE company;

Step 5: Refresh the Navigator

Refresh the Schemas panel to view the newly created database.

Troubleshooting

Problem 1: MySQL Service Fails to Start

Check service status: 

brew services info mysql

Attempt to start manually: 

mysql.server start

Problem 2: Port 3306 Already in Use

Check which process is using the MySQL port: 

sudo lsof -i :3306

Example output: 

mysqld  1234  user  TCP *:mysql (LISTEN)

Problem 3: Access Denied for User

You may see: 

ERROR 1045 (28000):
Access denied for user 'root'@'localhost'

Verify the username and password used for the connection.

Problem 4: Unknown Database

You may see: 

ERROR 1049 (42000):
Unknown database 'company'

Verify that the database exists: 

SHOW DATABASES;

Create the database if necessary: 

CREATE DATABASE company;
