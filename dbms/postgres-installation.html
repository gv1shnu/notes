Installing PostgreSQL on macOS

Difficulty: Easy

In this guide, PostgreSQL is installed and accessed through the terminal using the psql client.

Step 1: Verify Homebrew Installation

Check whether Homebrew is installed: 

brew --version

If Homebrew is not installed, install it using: 

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Step 2: Install PostgreSQL

Install PostgreSQL using Homebrew: 

brew install postgresql@18

Verify the installation: 

psql --version

Example output: 

psql (PostgreSQL) 18.4

Step 3: Start PostgreSQL Service

Start PostgreSQL as a background service: 

brew services start postgresql@18

Check the service status: 

brew services list

Expected output: 

postgresql@18 started

Step 4: Connect to PostgreSQL

Connect using: 

psql -h localhost -U <username> -d postgres

Example: 

psql -h localhost -U vishnu -d postgres

If the connection succeeds, the prompt changes to: 

postgres=>

Step 5: View Existing Databases

\l

This command lists all available databases.

Step 6: View Existing Roles

\du

This command lists all database users (roles).

Step 7: Create a New Database

CREATE DATABASE company;

Connect to the newly created database: 

\c company

Step 8: Verify the Current Database

SELECT current_database();

Step 9: Exit PostgreSQL

\q

Important Note

PostgreSQL requires both a valid user (role) and a valid database. 

psql -U vishnu -d company

Here:

  • vishnu is the user (role).
  • company is the database.

If the database name is omitted, PostgreSQL automatically attempts to connect to a database with the same name as the user.

Troubleshooting

Problem 1: PostgreSQL Service Fails to Start

The service status may show: 

postgresql@18 error 1

Check the service details: 

brew services info postgresql@18

Attempt to start PostgreSQL manually: 

/opt/homebrew/opt/postgresql@18/bin/postgres -D /opt/homebrew/var/postgresql@18

Problem 2: Port 5432 Already in Use

PostgreSQL may display: 

could not bind IPv4 address "127.0.0.1": Address already in use
could not bind IPv6 address "::1": Address already in use
FATAL: could not create any TCP/IP sockets

This indicates that another PostgreSQL server is already running on port 5432.

Check which process is using the port: 

sudo lsof -i :5432

Example output: 

postgres 3989 postgres ... TCP *:postgresql (LISTEN)

Identify the PostgreSQL installation: 

sudo ps -fp 3989

Problem 3: Role Does Not Exist

Attempting to connect may produce: 

psql -h localhost -U postgres
FATAL: role "postgres" does not exist

This means the PostgreSQL cluster does not contain a role named postgres.

Use an existing role instead: 

psql -h localhost -U vishnu -d postgres

Problem 4: Database Does Not Exist

Attempting to connect may produce: 

psql -h localhost -U vishnu
FATAL: database "vishnu" does not exist

PostgreSQL assumes: 

Database Name = User Name

when the database is not specified.

Specify an existing database explicitly: 

psql -h localhost -U vishnu -d postgres
