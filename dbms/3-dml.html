DML — Data Manipulation Language

DML commands manipulate data inside tables.

INSERT

INSERT adds new rows into a table. The DBMS controls all writes so it can enforce constraints, maintain indexes, and log changes. You can't write directly to the storage file/database.

MySQL / PostgreSQL


INSERT INTO players
VALUES (1, 'TenZ', 'Radiant', 842);

After insertion:

player_id username rank rr
1 TenZ Radiant 842

UPDATE

UPDATE modifies existing rows. You could always remove the existing row and add the updated row as a separate. However, UPDATE is atomic, it preserves row position for indexes, and it's less verbose.

MySQL / PostgreSQL


UPDATE players
SET rr = 900
WHERE player_id = 1;

After update:

player_id username rank rr
1 TenZ Radiant 900

DELETE

DELETE removes one or more rows.

MySQL / PostgreSQL


DELETE FROM players
WHERE player_id = 1;

After deletion:

player_id username rank rr

Note:
Running DELETE without WHERE removes all rows from the table. 


DELETE FROM players;
