SQL Datatypes

Without type enforcement, nothing stops someone from inserting "hello" into a salary column, or "abc" into a player ID. Each column is assigned a datatype, and the DBMS rejects any value that doesn't match. This is called a domain constraint, and it is the first line of defence for the integrity problems we saw earlier in intro.

Choosing proper datatypes also improves storage efficiency and data validation.

Common Datatypes (MySQL + PostgreSQL)

Datatype Purpose Example
INT Whole numbers 842
DECIMAL(10, 2) Exact decimals (e.g. money) 45000.00
FLOAT Approximate decimals 3.14159
CHAR(5) Fixed-length text A1234
VARCHAR(30) Variable-length text TenZ
TEXT Large / unbounded text a long match summary
JSON Structured JSON documents {"player":"Alice","score":42}
BOOLEAN Stores true or false true
DATE Stores dates 2026-05-27

Note: FLOAT stores numbers as approximate binary floating point, so two values that look equal may not compare as equal. For money, rates, or anything you compare or sum exactly, use DECIMAL (also called NUMERIC) instead.

BOOLEAN is worth one caveat too: PostgreSQL has a true boolean type, while MySQL stores it as TINYINT(1) under the hood.

MySQL-specific Datatypes

Datatype Purpose
TINYINT Small integer values
MEDIUMINT Medium-sized integer values
ENUM Restricts values to a predefined list
AUTO_INCREMENT Automatically generates IDs 
tier ENUM('Iron', 'Bronze', 'Silver')

Note: AUTO_INCREMENT is strictly a column attribute, not a datatype on its own — it is attached to an integer column so the DBMS fills in the next ID automatically.

PostgreSQL-specific Datatypes

Datatype Purpose
SERIAL Auto-generated integer IDs
BIGSERIAL Large auto-generated IDs
UUID Universally unique identifiers
JSONB Efficient binary JSON storage
ARRAY Stores arrays of values 
player_id SERIAL PRIMARY KEY

Note: SERIAL isn't a real stored type — it's shorthand that creates an integer column backed by an auto-incrementing sequence.

Also, both MySQL and PostgreSQL have a JSON type; only PostgreSQL adds the binary JSONB variant, which is faster to query.
