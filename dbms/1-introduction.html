Introduction

The human brain's memory capacity in the average adult can store 2.5PB. [1] However, the efficiency of the stored information for analysis is often clumsy, because people forget. So, we started writing in books.

It's still considered the best and most private way to keep information stored someplace, but not optimal. Books were good for storing information, but bad for managing large amounts of it. Problem: They can get lost, burned, wet, torn, thrown, stolen, etc.

When computers came into existence, people started to outsource storing information to the machine and replaced books with disks.
Solution: Disks introduced faster reading and writing, easier copying, easier searching, cheaper long-term storage, information could be shared quickly.


One way to keep information on a computer is to store it in permanent files. [2] The system for saving, editing, and deleting the files was called file-based system, named appropriately.

Organisations used the flat-file with structs dumping to and reading from the file. Structs written directly to disk usually avoided pointers because memory addresses are not portable across program executions. Example, Employee struct:

This meant that writing was easy: dump complete struct to disk. Reading could be simple when record locations were known and records had fixed sizes. [3] The hard parts were searching, size enforcement (when number exceeded range), upgrades (when structs were changed), OS incompatibilities (linux/windows).

File-based systems became difficult to manage as data volume, complexity, and concurrent users increased.

The following challenges in file-based system were noted, and consequently DBMS was introduced solving them.

    The same information being kept in several different places, as different individuals may required to keep copy of certain data.

The difficulties that arise from using the file-based system prompted the development of a new approach for managing information, called the database approach. Interestingly, databases are still stored as files on disk in most systems. What changed is not the storage medium itself, but the way data is organized and managed. The software built to manage this database is called, Database management system (DBMS). A DBMS sits between the user and the files, providing structure, security, consistency, and efficient access to the stored data.

Storage technology evolved from paper records to magnetic tapes, hard disks, solid-state drives, and cloud storage. Modern DBMS software sits on top of these storage devices and provides an organized way to store, retrieve, and manage data.

Formally, a database is a collection of related data. [4] Conceptually, a database table can be viewed as an organized collection of records similar to data stored in file-based systems.

The database definition or descriptive information is stored by the DBMS in the form of a database catalog or dictionary; it is called meta-data. DBMS handles concurrent accesses to the database, enforces constraints, manages permissions, maintains consistency across all changes. A DBMS can also handle several databases, keep secure backups, insulate actual data from programs, and so much.

With modern technology, data is no longer confined to local files on a single computer. Cloud-based services allow us to store documents securely, access them from anywhere, and analyse large amounts of information in real time.

For example, platforms such as DigiLocker enable users to store and retrieve important documents online, while analytics services can track and visualise detailed statistics and performance metrics.
