Introduction

The human brain's memory capacity in the average adult can store around 2.5PB. [1] However, the efficiency of the stored information for analysis is often clumsy, because people forget. So, we started writing in books.

It's still considered the best and most private way to keep information stored someplace, but not optimal for complex data. Books were good for storing information, but bad for managing large amounts of it.
Problem: They can get lost, burned, wet, overwritten, torn, thrown, stolen, etc.

When computers came into existence, people started to outsource storing information to the machine and replaced books with disks.
Solution: Disks introduced faster reading and writing, easier copying, easier searching, cheaper long-term storage, information could be shared quickly.

One way to keep information on a computer is to store it in permanent files. [2] The system for saving, editing, and deleting the files was called file-based system, named appropriately.

Organisations used the flat-file with structs dumping to and reading from the file. Structs written directly to disk usually avoided pointers because memory addresses are not portable across program executions. Example, Employee struct: 

C

struct Employee {
    int id;
    char name[50];
    unsigned int salary;
    char dept[20];
};
Flat file on floppy disk

C

struct Employee e;
FILE *fp = fopen("employees.dat", "rb");

while (fread(&e, sizeof(struct Employee), 1, fp) == 1) {
    if (e.id == 101) {
        printf("%d %s %u %s\n", e.id, e.name, e.salary, e.dept);
        break;
    }
}

fclose(fp);

C

struct Employee e = {
    .id = 101,
    .name = "John",
    .salary = 50000,
    .dept = "HR"
};

FILE *fp = fopen("employees.dat", "ab");

fwrite(&e, sizeof(struct Employee), 1, fp);

fclose(fp);
This meant that writing was easy: dump complete struct to disk. Reading could be simple when record locations were known and records had fixed sizes. [3] The hard parts were searching, upgrades (when structs were changed), endianness (BE/LE), struct padding/alignment, and primitive type sizes (int/long differ by platform), etc.
