What exactly is a DBMS?
A Database Management System (DBMS) is software that manages databases.

As mentioned previously, the conceptual evolution from structs to tables is natural. Understanding DBMS terminology is important before moving further.

DBMS Terminology

Term Meaning Example
Table / Relation Collection of related data Employees
Row / Record / Tuple One complete entry E101, Rahul, HR
Column / Attribute One property of data EmployeeName
Primary Key Uniquely identifies a row EmpID
Foreign Key References another table DeptID
Schema Structure of the database Tables + columns + constraints
Constraint Rule enforced on data Salary > 0
Index Speeds up searching Search EmpID quickly
Query Request for data Find all employees in HR

As tables are also called relations, databases that store data using tables are called Relational Databases.

A Relational Database Management System (RDBMS) manages relational databases.

The most common language used to interact with an RDBMS is: SQL (Structured Query Language).

Popular SQL DBMS:

  • MySQL
  • PostgreSQL

SQL Command Categories

Abbreviation Full Form Purpose Common Commands
DDL Data Definition Language Defines database structure CREATE, ALTER, DROP
DML Data Manipulation Language Manipulates data INSERT, UPDATE, DELETE
DQL Data Query Language Queries data SELECT
DCL Data Control Language Controls permissions GRANT, REVOKE
TCL Transaction Control Language Manages transactions COMMIT, ROLLBACK

We'll start with DDL because databases must be structured before storing data.

How do we create a database?


MySQL / PostgreSQL

CREATE DATABASE company;

How do we use the database?

We need to mention which database we are going to be storing the table in. For that, we use the following statements: 


MySQL

USE company;




PostgreSQL

\c company;

How do we create a table?

A table stores related data in rows and columns. Now, let's create a table.

MySQL / PostgreSQL


    
CREATE TABLE players (
    player_id INT,
    username VARCHAR(30),
    rank VARCHAR(20),
    rr INT
);

This creates a table named players.

player_id username rank rr
1 TenZ Radiant 842

Each column has a datatype. What datatypes are there? Let's discuss.

Datatype Purpose Example
INT Stores integers 842
VARCHAR(30) Stores variable-length text TenZ
BOOLEAN Stores true or false true
FLOAT Stores decimal numbers 1.27
DATE Stores dates 2026-05-27

Choosing proper datatypes improves storage efficiency and data validation.
