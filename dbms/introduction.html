Introduction

The human brain's memory capacity in the average adult can store 2.5PB. [1] However, the efficiency of the stored information is often clumsy, depending on individual. So, we started writing. It's still considered the best and most private way to keep information stored someplace, but not the efficient. Books can get lost, burned, wet, torn, thrown, stolen, etc.

When computers came into existence, people started to outsource storing information to the machine and replaced books with disks. One way to keep information on a computer is to store it in permanent files. [2] The system for saving, editing, and deleting the files was called file-based system, named appropriately.

Organisation used the flat-file with structs dumping to file. Structs would not have pointers, but contain real data. This meant that writing was easy: dump complete struct to disk. This also meant that reading was easy: do file seek and read struct. [3] The hard parts were searching, size enforcement (when number exceeded range), upgrades (when structs were changed), os incompatibilities (linux/windows). This didn't last longer than was expected, as there were so many demerits. The following challenges were noted.

  1. Data redundancy.

    The same information being kept in several different places, as different individuals may required to keep copy of certain data.

  2. Data inconsistency.

    A situation where various copies of the same data are conflicting, wastes storage space and duplicates effort.

  3. Integrity problems.

    It is difficult to make changes to the application programs in order to enforce new constraints.

  4. Concurrency problems.

    Typically, in a file-based system, when an application opens a file, that file is locked. This means that no one else has access to the file at the same time.

The difficulties that arise from using the file-based system have prompted the development of a new approach in managing large amounts of organizational information called the database approach. The software built to manage this database, named, Database management system (DBMS).

Formally, a database is a collection of related data. [4] The database definition or descriptive information is stored by the DBMS in the form of a database catalog or dictionary; it is called meta-data. DBMS handles concurrent accesses to the database, enforces constraints, manages permissions, maintains consistency across all changes. A DBMS can also handle several databases, keep secure backups, insulate actual data from programs, and so much.
