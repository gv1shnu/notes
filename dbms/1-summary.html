Summary of transition from file-based system to DBMS.

Difficulty: Easy

The difficulties that arise from using the file-based system prompted the development of a new approach for managing information, called the database approach. Interestingly, databases are still stored as files on disk in most systems. What changed is not the storage medium itself, but the way data is organized and managed. The software built to manage this database is called, Database management system (DBMS). A DBMS sits between the user and the files, providing structure, security, consistency, and efficient access to the stored data.

Storage technology evolved from paper records to magnetic tapes, hard disks, solid-state drives, and cloud storage. Modern DBMS software sits on top of these storage devices and provides an organized way to store, retrieve, and manage data.

Formally, a database is a collection of related data in the form of tables. Conceptually, a database table can be viewed as an organized collection of records similar to data stored in file-based systems.

The database definition or descriptive information is stored by the DBMS in the form of a database catalog or dictionary; it is called meta-data. DBMS handles concurrent accesses to the database, enforces constraints, manages permissions, maintains consistency across all changes. A DBMS can also handle several databases, keep secure backups, insulate actual data from programs, and so much more.

With modern technology, data is no longer confined to local files on a single computer. Cloud-based services allow us to store documents securely in distributed systems, access them from anywhere, and analyse large amounts of information in real time.

For example, platforms such as DigiLocker enable users to store and retrieve important documents online, while analytics services can track and visualise detailed statistics and performance metrics.

Summary

  • Problem: Unreliable record keeping on books.
    Solution: Computer. Files.
  • Problem: Inefficient file handling.
    Solution: Databases. DBMS.
