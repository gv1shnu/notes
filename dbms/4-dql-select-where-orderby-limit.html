DQL — Data Query Language

DQL commands query data from tables.

SELECT

SELECT reads data from a table.

Current table:

player_id username rank rr
1 TenZ Radiant 842
2 aspas Immortal 611
3 yay Ascendant 402

MySQL / PostgreSQL


SELECT username, rank
FROM players;

Output:

username rank
TenZ Radiant
aspas Immortal
yay Ascendant

Note: SELECT * returns everything, which is wasteful on large tables and couples your query to the table's structure.

WHERE

WHERE filters rows.

Current table:

player_id username rank rr
1 TenZ Radiant 842
2 aspas Immortal 611
3 yay Ascendant 402

MySQL / PostgreSQL


SELECT *
FROM players
WHERE rank = 'Radiant';

Output:

player_id username rank rr
1 TenZ Radiant 842

ORDER BY

ORDER BY sorts rows.
DESC = descending order.
ASC = ascending order.

Current table:

username rr
TenZ 842
aspas 611
yay 402

MySQL / PostgreSQL


SELECT *
FROM players
ORDER BY rr DESC;

Output:

username rr
TenZ 842
aspas 611
yay 402

LIMIT

LIMIT restricts the number of rows returned. Problem: A players table in production might have 5 million rows. SELECT * FROM players would return all 5 million. That crashes slow clients, wastes bandwidth, and locks the database longer.
Solution: LIMIT returns only the first N rows. This is also used for pagination.

Current table:

username rr
TenZ 842
aspas 611
yay 402

MySQL / PostgreSQL


SELECT *
FROM players
LIMIT 2;

Output:

username rr
TenZ 842
aspas 611
