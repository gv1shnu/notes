3. Integrity problems.

It is difficult to make changes to the application programs in order to enforce new constraints.

EmpID EmployeeName Phone Salary
E101 Amit Singh 9876543210 45000
E102 Neha Joshi 98AB112233 52000
E101 Vikram Rao 9898989898 47000
E104 Sneha Iyer 9812345678 -15000

DBMS Solution
Constraint Purpose
PRIMARY KEY No duplicate EmpID
CHECK No negative salary
NOT NULL Mandatory value required
UNIQUE No duplicate values allowed
FOREIGN KEY Maintains valid references between tables
DEFAULT Assigns a value when none is provided

DBMS enforces rules automatically.

5. Concurrency problems.

Typically, in a file-based system, when an application opens a file, that file is locked. This means that no one else has access to the file at the same time.


DBMS Solution
User Operation
User A Update salary
User B Read employee data

DBMS manages multiple users safely using transactions and locks.
