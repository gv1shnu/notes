Dictionaries, Sets & Tuples

Difficulty: Medium

Three more built-in containers, each with a distinct job. Pick by the question you're asking:

Type Shape Use when
dict key → value pairs you look things up by a label
set unordered unique values you care about membership / dedup
tuple ordered, immutable sequence a fixed group that shouldn't change

Dictionary — lookups by key

player = {"name": "TenZ", "score": 842}
player["name"]          # 'TenZ'
player["level"] = 99      # add / update a key
player.get("rank", "NA")  # safe read with a default
"score" in player        # True — checks keys

for k, v in player.items():  # iterate key+value
    print(k, v)

Keys are unique and lookups are O(1) average (it's a hash table). Reading a missing key with [] raises an error — use .get(key, default) when absence is normal.

A dict doesn't scan — it computes where a key lives. The key is run through hash() to pick a bucket, and the value is read straight from that slot. No matter how many keys exist, it's one jump — that's the O(1) average.

Set — membership & uniqueness

seen = {1, 2, 2, 3}        # {1, 2, 3} — duplicates collapse
3 in seen                  # True (O(1))
seen.add(4)
{1, 2} & {2, 3}            # {2}  intersection
{1, 2} | {2, 3}            # {1, 2, 3} union
{1, 2} - {2}               # {1}  difference

A one-line dedup: list(set(items)). Sets are unordered, so don't index them — they're for "is this in here?" and the algebra of groups.

Tuple — fixed and unpackable

point = (3, 4)
x, y = point          # unpacking: x=3, y=4
point[0] = 9          # TypeError — tuples can't change

Because they're immutable, tuples can be used as dictionary keys (a list can't). They're also what's quietly happening when you write a, b = b, a to swap, or return x, y from a function — that's tuple packing.
