Inheritance & Special Methods

Two ways classes get powerful: one class can build on another, and any class can hook into Python's built-in syntax.

Inheritance — build on what exists

A subclass inherits everything from its parent, then adds or overrides. Don't rewrite the shared parts — extend them. 

class Player:
    def __init__(self, name):
        self.name = name
    def role(self):
        return "player"

class Captain(Player):          # Captain IS-A Player
    def __init__(self, name, team):
        super().__init__(name)   # run the parent's setup
        self.team = team
    def role(self):             # override
        return "captain"

c = Captain("TenZ", "SEN")
c.name      # 'TenZ'      inherited
c.role()    # 'captain'   overridden

super().__init__(...) calls the parent's constructor so you don't duplicate its work; then the child adds its own. Overriding role means a Captain answers differently from a plain Player — same method name, specialized behaviour.

Special ("dunder") methods

Methods wrapped in __double_underscores__ let your objects plug into Python's own syntax — printing, len(), +, ==. Define them and your class behaves like a built-in.

Define Powers
__init__ Player(...) — construction
__str__ print(p) — the human-readable form
__len__ len(p)
__eq__ p == q
__add__ p + q 
class Vec:
    def __init__(self, x, y):
        self.x, self.y = x, y
    def __add__(self, other):       # defines the + operator
        return Vec(self.x + other.x, self.y + other.y)
    def __str__(self):
        return f"({self.x}, {self.y})"

print(Vec(1, 2) + Vec(3, 4))   # (4, 6)

That's operator overloading: + on two Vecs now means vector addition, and print shows a clean (4, 6) instead of a cryptic memory address. Same syntax you already use on numbers and lists — now on your own types.
