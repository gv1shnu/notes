Strings

Text, written in single, double, or triple quotes. The defining fact: strings are immutable — every "edit" returns a new string, the original never changes. They're also sequences, so indexing and slicing work just like a list of characters. 

s = "Hello"
s = 'also fine'
s = """spans
multiple lines"""

Indexing & slicing

s = "Python"
s[0]      # 'P'   first
s[-1]     # 'n'   last (negative = from the end)
s[0:3]    # 'Pyt' [start:stop) — stop excluded
s[2:]     # 'thon'
s[::-1]   # 'nohtyP' reversed (step of -1)
len(s)    # 6

Slicing is [start:stop:step], and stop is always excluded — s[0:3] gives three characters, indices 0,1,2. That half-open convention shows up everywhere in Python.

Building strings

a = "foo" + "bar"     # 'foobar' concatenation
b = "ab" * 3           # 'ababab' repetition

name, score = "TenZ", 842
print(f"{name} has {score}")   # f-string: 'TenZ has 842'

The f-string (note the f prefix) is the modern way to interpolate — drop any expression inside { } and it's formatted in place. Far cleaner than +-ing pieces together.

The methods you'll reach for

Call Does
s.upper() / s.lower() change case (returns a new string)
s.strip() trim surrounding whitespace
s.split(",") string → list of pieces
",".join(parts) list of strings → one string
s.replace("a", "b") swap substrings
"x" in s membership test (True/False)

split and join are inverses and do most of the heavy lifting in real text work. Because strings are immutable, all of these return a result — s.upper() alone changes nothing unless you assign it.
