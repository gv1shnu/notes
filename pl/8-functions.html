Functions

Difficulty: Easy

Loops repeat a block in place; a function lets you name that block and reuse it anywhere — the unit of "don't repeat yourself." Define it with def, feed it parameters, and get something back with return. 

def greet(name, greeting="Hello"):   # greeting has a default
    return f"{greeting}, {name}"

greet("TenZ")                  # 'Hello, TenZ'
greet("TenZ", "Hi")            # 'Hi, TenZ'      positional
greet(name="TenZ", greeting="Yo")  # by keyword (order-free)

A function with no return hands back None. Default values make arguments optional; keyword arguments let callers name what they pass, so order stops mattering and the call reads clearly.

Taking any number of arguments

def total(*nums):          # *args  -> a tuple of extras
    return sum(nums)

total(1, 2, 3)             # 6

def config(**opts):        # **kwargs -> a dict of named extras
    return opts

config(debug=True, level=5)  # {'debug': True, 'level': 5}

Scope: local vs global

count = 0              # global

def bump():
    count = 99         # a NEW local — the global is untouched

bump()
print(count)          # still 0

Names assigned inside a function are local — they vanish when it returns and don't leak out. A function can read an outer variable, but assigning creates a fresh local unless you explicitly say global (rarely a good idea — prefer returning a value).

Talking to the user

name = input("Your name? ")   # input() always returns a STRING
age  = int(input("Age? "))  # cast it for math
print(f"Hi {name}, next year you're {age + 1}")

input() hands back text every time — even "25" arrives as "25", so wrap it in int()/float() before doing arithmetic (the same cast from numbers).
