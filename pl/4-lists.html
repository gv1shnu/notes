Lists

The workhorse container: an ordered, mutable sequence that can hold anything, even a mix of types. Indexing and slicing work exactly like strings — the difference is you can change a list in place. 

nums = [5, 2, 9, 1]
nums[0]       # 5
nums[-1]      # 1
nums[1:3]     # [2, 9]   slicing returns a new list
nums[0] = 99  # mutable: now [99, 2, 9, 1]

Growing and shrinking

Call Does
nums.append(7) add to the end (O(1))
nums.insert(0, 7) add at an index (O(n) — shifts the rest)
nums.pop() remove + return the last (or pop(i))
nums.remove(9) delete the first matching value
nums.extend([8, 6]) append all of another list
nums.sort() / nums.reverse() reorder in place

List comprehensions

The Pythonic way to build a list from another — a loop folded into one readable line: 

squares = [x * x for x in range(5)]          # [0, 1, 4, 9, 16]
evens   = [x for x in nums if x % 2 == 0]   # keep only evens

Read it as "this expression, for each item, optionally if a condition." Once it clicks, you'll write these instead of three-line loops constantly.

The gotcha: lists are shared by reference

a = [1, 2, 3]
b = a           # b is the SAME list, not a copy
b.append(4)     # a is now [1, 2, 3, 4] too!

c = a[:]        # a real copy (slice) — or list(a)

Assigning a list doesn't duplicate it — both names point at the same object, so mutating one is seen through the other. When you need an independent copy, slice it (a[:]) or call list(a). This bites everyone exactly once.
