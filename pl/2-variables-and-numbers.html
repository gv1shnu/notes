Variables & Numbers

Difficulty: Easy

Code runs; now it needs somewhere to keep values. A variable is just a name pointing at a value. No declaration, no type keyword — you assign, and Python figures out the rest. Reassign it to something of a different type and Python won't blink. 

name = "Vishnu"     # a string
score = 842          # an int
score = "high"       # now a string — perfectly legal

Names are lowercase with underscores (player_score), can't start with a digit, and are case-sensitive. The = binds the name to the object on its right.

The number types

Type Example Note
int 842 arbitrary precision — no overflow, ever
float 3.14 fractional; subject to rounding (0.1 + 0.2 != 0.3)
complex 2 + 3j real + imaginary, built in

Arithmetic

7 + 2     # 9
7 - 2     # 5
7 * 2     # 14
7 / 2     # 3.5   true division -> always float
7 // 2    # 3     floor division
7 % 2     # 1     remainder (modulo)
7 ** 2    # 49    power

The one that trips people: / always gives a float (4 / 2 is 2.0); use // when you want an integer result. Mixing an int and a float promotes to float.

Checking and converting

type(score)   # <class 'int'>
int("42")    # 42    string -> int
float(3)     # 3.0
str(842)     # "842" number -> string

Conversions are explicit on purpose — Python won't quietly add a string to an int. The bridge between "user typed text" and "math" is one of these casts, usually int(...) or float(...).
