Conditionals & Booleans

Code that makes decisions needs something to decide on: a boolean, either True or False. Comparisons produce them, logic combines them, and if acts on them.

Producing booleans

5 == 5     # True   equal (two = signs!)
5 != 3     # True   not equal
5 > 3      # True
5 <= 5     # True

(5 > 3) and (2 > 9)   # False  both must hold
(5 > 3) or  (2 > 9)   # True   either holds
not (5 > 3)            # False  flip it

The classic bug: = assigns, == compares. Python uses the words and, or, not rather than symbols — reads like English on purpose.

Truthiness

Any value can be tested directly — you don't always need an explicit comparison. "Empty" things are falsy:

Falsy Truthy
0, 0.0, "", [], {}, None, False everything else (any non-zero number, any non-empty container)

So if items: reads as "if the list has anything in it" — cleaner than if len(items) > 0:.

if / elif / else

score = 842
if score >= 800:
    rank = "Radiant"
elif score >= 500:
    rank = "Immortal"
else:
    rank = "Diamond"

Python checks branches top to bottom and runs the first that's true, then skips the rest. The indentation (4 spaces) is what marks the body — there are no braces, so misaligned code is a real bug, not a style nit. For a quick one-liner there's the ternary: 

label = "pass" if score >= 500 else "fail"
