Metadata

Difficulty: Intermediate

Every digital file contains information about itself in addition to its contents. This information is known as metadata. While users often focus on filenames and folders, metadata provides important details about when, where and how a file was created or modified.

Good archive management depends on understanding metadata because it improves searching, sorting, verification and long-term preservation.

What is Metadata?

File

        ├── Name

Metadata describes a file. It is not usually the information stored inside the file itself, but information about that file.

Common Metadata

Metadata Description
Filename Human-readable identifier.
File Size Storage occupied by the file.
Creation Time When the file was created.
Modified Time Most recent modification.
File Type PDF, JPEG, MP4, ZIP, etc.
Permissions Who may read or modify the file.

File Timestamps

Operating systems commonly maintain multiple timestamps for files.

TimestampPurpose
CreatedWhen the file first appeared.
ModifiedWhen contents last changed.
AccessedWhen the file was last opened.

These timestamps help sort archives chronologically and identify recently modified data.

Metadata Is Not Always Reliable

Metadata can be edited, lost or regenerated when files are copied between devices or converted into different formats.

  • ZIP archives may change timestamps.
  • Image editors may remove EXIF information.
  • Cloud services may modify metadata during synchronization.
  • Copying files can change creation dates.

Metadata vs File Contents

Metadata

"What is this file?"

↓

Contents

"What information does it contain?"

Both are valuable. Metadata helps manage an archive, while the contents provide the actual information you wish to preserve.

Key Takeaway

Metadata makes large collections searchable and manageable. Understanding timestamps, EXIF information, permissions and file attributes allows you to organize, preserve and share data more effectively while avoiding accidental information disclosure.
