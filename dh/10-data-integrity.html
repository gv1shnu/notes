Data Integrity

Difficulty: Intermediate

Storing data is only half the challenge. The other half is ensuring that the data remains correct and readable years later. Files can become corrupted without obvious warning due to failing hardware, transmission errors, software bugs or media degradation.

Data integrity is the practice of detecting corruption before it results in permanent information loss.

Identical Data

Integrity means that a file retrieved in the future is exactly the same as the file that was originally stored.

Silent Data Corruption

Not every storage failure is obvious. Sometimes a file changes without producing an error message. This is known as silent data corruption.

Possible CauseExample
Failing storage mediaUnreadable sectors.

SMART Monitoring

Many storage devices provide Self-Monitoring, Analysis and Reporting Technology (SMART) statistics that indicate drive health.

  • Reallocated sectors
  • Read errors
  • Power-on hours
  • Temperature
  • Pending sectors

Monitoring trends is often more useful than reacting only after complete failure.

Filesystem Scrubbing

Some filesystems periodically verify stored data by reading blocks and comparing them against stored integrity information. This process is commonly called scrubbing.

Scrubbing detects corruption that might otherwise remain unnoticed for years.

Key Takeaway

An archive should not simply be stored and forgotten. Regular integrity verification, health monitoring and checksum validation help ensure that valuable information remains unchanged and recoverable for many years.
