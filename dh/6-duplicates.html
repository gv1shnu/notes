Duplicates & Deduplication

Difficulty: Intermediate

As digital collections grow, duplicate files become inevitable. They waste storage, increase backup times, clutter search results and make archives more difficult to maintain. Understanding how duplicates are identified is an important step toward building an efficient digital archive.

What Is a Duplicate?

Not every similar-looking file is actually a duplicate.

TypeDescription
Exact DuplicateTwo files with identical contents.
Near DuplicateSimilar files with small differences.
VersionA newer or older revision of a document.
CopyA manually duplicated file with a different name.

Why File Names Are Not Enough

holiday.jpg
holiday (1).jpg

Different names do not necessarily mean different files. Likewise, two files with the same name may contain completely different data.

Checksums & Hashes

A checksum (or hash) is a short digital fingerprint calculated from a file's contents. If two files produce the same cryptographic hash, they are overwhelmingly likely to be identical.

Hashes are calculated from the data itself, not from the filename or location.

Near Duplicates

Near duplicates cannot usually be detected with ordinary hashes because their contents are not exactly the same.

  • Resized photographs.
  • Compressed versions of the same video.
  • Edited documents.
  • Different image quality levels.
  • Different audio bitrates.

These require similarity detection rather than exact matching.

Deduplication

Deduplication removes unnecessary copies while preserving a single copy of the data.

Hard Links vs Symbolic Links

TypePurpose
Hard LinkMultiple directory entries referencing the same file data.
Symbolic LinkA pointer to another file or directory.

These mechanisms reduce unnecessary duplication without creating additional copies of the underlying data.

Should Everything Be Deduplicated?

No. Different versions of a document or project often represent important history. Removing them blindly can permanently destroy useful information.

Key Takeaway

Deduplication is about reducing unnecessary repetition without losing valuable information. Exact duplicates are good candidates for removal, while versions and near duplicates should be reviewed carefully before deletion.
