Every device involved in this journey can potentially observe some information about your connection. They may not see the same information, but each contributes to the overall picture.

IP Addresses

An IP address is the network address assigned to a device. It allows data to be routed between computers across the Internet, much like a postal address allows physical mail to reach a destination. 

192.168.1.25  → Local (Private) Address

203.122.x.x   → Public Address

Your home router usually assigns private IP addresses to devices inside your local network while your ISP assigns a public IP address that websites on the Internet see.

HTTP vs HTTPS

HTTP sends web traffic in plaintext. HTTPS encrypts the communication between your browser and the website using TLS.

HTTP HTTPS
Unencrypted Encrypted
Easy to intercept on insecure networks. Protects data in transit.
Should be avoided. Standard for modern websites.

HTTPS protects the contents of your communication but does not hide the fact that a connection exists.

Key Takeaway

Every network request passes through several systems before reaching its destination. Understanding IP addresses, DNS and HTTPS provides the foundation required to understand VPNs, encrypted DNS, Tor and other privacy technologies.
