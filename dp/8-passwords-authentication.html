Passwords & Authentication

Difficulty: Easy

Passwords remain the first line of defence for most online accounts. Unfortunately, weak passwords, password reuse, and poor authentication practices continue to be among the most common causes of account compromise.

The goal is not to create passwords that you can remember easily. The goal is to make them difficult to guess, unique for every account, and easy to manage using the right tools.

Why Password Reuse Is Dangerous

Imagine you use the same password for ten websites.

This technique is known as credential stuffing. Attackers use previously leaked username and password combinations against unrelated services because many people reuse passwords.

Characteristics of a Good Password

  • Unique for every account.
  • Long rather than merely complex.
  • Random or generated.
  • Never reused.
  • Stored in a password manager instead of memorised.

Length generally contributes more to password strength than replacing letters with symbols or numbers.

Password Managers

A password manager generates, stores and autofills unique passwords for each account. Instead of remembering hundreds of passwords, you only need to protect one strong master password. 

Master Password
        │
        ▼
Password Vault
        │
        ├── Email
        ├── Banking
        ├── Shopping
        ├── Gaming
        └── Work

Because every account receives a different password, a breach affecting one website does not automatically compromise the others.

A Concrete Choice: Bitwarden

Bitwarden is an open-source password manager, and that openness is its defining trait: because the source code is public, its security can be independently audited rather than taken on trust. For most people it is the sensible default.

Its specialities:

  • Zero-knowledge encryption. Your vault is encrypted and decrypted on your own device, using a key derived from your master password. The servers only ever hold ciphertext, so Bitwarden itself cannot read your vault — and neither can an attacker who breaches their servers.
  • Works everywhere, free. Desktop apps, browser extensions, mobile apps and a command-line tool all sync the same vault. The free tier covers unlimited passwords across unlimited devices, which is unusual among competitors.
  • Self-hostable. You can run your own Bitwarden server (the lightweight community implementation, Vaultwarden, is popular) so the vault never leaves infrastructure you control. This is the option privacy-conscious users reach for.

The catch: the master password is a single point of failure. It is stored nowhere and cannot be reset — forget it and the vault is unrecoverable, by design. Protect the vault itself with MFA, and keep any emergency recovery material offline and separate.

Multi-Factor Authentication (MFA)

Authentication factors are commonly grouped into three categories:

  • Something you know — Password or PIN.
  • Something you have — Security key or authenticator app.
  • Something you are — Fingerprint or facial recognition.
Password
        +
Authenticator App
        ↓
Two-Factor Authentication

Using more than one factor significantly reduces the risk posed by stolen passwords.

SMS vs Authenticator Apps

Method Comments
SMS Codes Better than no MFA, but vulnerable to SIM-swapping and interception.
Authenticator Apps Generate time-based codes locally and are generally preferred.
Hardware Security Keys Provide strong phishing resistance and are recommended for high-value accounts.

Security Questions

Many security questions have answers that are publicly available or easily guessed. Treat them like additional passwords rather than truthful biographical information. Store those answers in your password manager if necessary.

Key Takeaway

Strong account security is achieved through unique passwords, a password manager and multi-factor authentication. Together they greatly reduce the impact of password leaks, credential stuffing and phishing attacks.
