Resources & Further Reading

Difficulty: Easy

This course intentionally focuses on concepts rather than recommending specific software. Technologies evolve, products change ownership, and best practices improve over time. Understanding the underlying principles allows you to evaluate new tools critically instead of relying on static recommendations.

Internet Standards (RFCs)

Many Internet technologies are documented through Request for Comments (RFC) documents published by the Internet Engineering Task Force (IETF).

  • HTTP
  • HTTPS / TLS
  • DNS
  • IPv4 & IPv6
  • SMTP, IMAP and POP3

Standards Organizations

  • Internet Engineering Task Force (IETF)
  • World Wide Web Consortium (W3C)
  • National Institute of Standards and Technology (NIST)
  • Open Web Application Security Project (OWASP)
  • Electronic Frontier Foundation (EFF)

Recommended Reading

  • NIST Cybersecurity Framework (CSF)
  • NIST Password Guidelines (SP 800-63B)

  • OWASP Cheat Sheet Series

Hands-on Learning

Reading is valuable, but practical experience develops a deeper understanding. Consider exploring:

  • Inspect HTTP requests and responses using your browser's Developer Tools.
  • Observe DNS lookups with networking utilities.
  • Experiment with browser profiles and cookie isolation.
  • Review application permissions on your computer and phone.
  • Practice restoring files from backups.
  • Create and test a password manager vault.

Stay Current

Privacy and security are continuously evolving fields. New standards, attack techniques, regulations and technologies appear every year. Revisit your privacy strategy periodically and verify information against current documentation rather than assuming guidance remains valid indefinitely.
