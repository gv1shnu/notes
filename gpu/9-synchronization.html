9. Synchronization.

The shared tool behind the solutions above: lock access to data or code so threads take turns. Use with care — synchronize everything and you create dead/livelock.

Lock Semaphore
Access one thread at a time up to N at a time
Analogy a ticket counter airline boarding groups
Use when only one may touch it a few may share safely

There are rarely bulletproof fixes for concurrency. Aim for a reasonable one: prioritize coherent data access, and decide how much under/over-utilization you can accept.
