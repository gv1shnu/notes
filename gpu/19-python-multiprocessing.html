Parallel syntax — Python 3

19. multiprocessing.

Beyond threads: run code in separate processes — even on a remote machine. Three ways to create one:

Method What it does
spawn fresh Python interpreter (most common)
fork a copy of the current process
forkserver a server that forks new processes on demand, cleans up dead ones
Tool Use
Process(target, args) build a process; start() to run, join() to wait for it
Queue / Pipe communicate between processes (queue = sequential, pipe = two-way)
Value / Array share one value, or a collection of primitives
Pool(n).map(fn, data) a worker pool runs fn across the data in parallel

Locks work here too, just across processes instead of threads.

In code 
Python

from multiprocessing import Pool

def square(x):
    return x * x

if __name__ == "__main__":
    with Pool(4) as p:
        print(p.map(square, [1, 2, 3, 4]))   # [1, 4, 9, 16]
