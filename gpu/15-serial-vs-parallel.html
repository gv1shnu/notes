Choosing an approach

Difficulty: Medium

15. Serial vs parallel.

Same task — find a value in a set — three ways. The right one depends on the data and how often you search.

Approach Cost Needs sort? Note
Serial linear O(n) no simplest; scans everything
Binary (divide & conquer) O(log n) yes worth it if you search far more than you sort
Parallel (slice across threads) scales with threads no with enough threads (GPU) → near-constant

Same eight units of work. Serial does them end-to-end — wall-clock grows with the whole list. Four threads do two each at the same time, so the wall-clock collapses to a quarter. The total work is identical; only the elapsed time changes — and only if you actually have the threads.

Parallel only wins when threads are many; managing and stopping them is the hard part.
