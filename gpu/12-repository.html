Concurrency patterns

Difficulty: Medium

12. Repository.

For state shared across processes. Each process runs on its own and asks a central repository when it needs to read or change shared state.

Who Responsible for
Repository keeping shared data atomic
Process managing its own state

Trade-off: hit the repository often and risk staleness; hit it rarely and a process may overwrite a newer state with old work.
