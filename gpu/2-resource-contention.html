Pitfalls of shared state

2. Resource contention.

Like a race condition, but about memory access and more asynchronous — it can even span separate machines.

Two threads each read the same DB row, add 1, and write back. Expected after both: 12.
Thread A Thread B stored value
read 10 10
read 10 10
write 11 11
write 11 11

One increment vanished. With many threads and shared rows, lost updates pile up.


Solution
Mechanism Effect
Transaction Read + write commit as one unit
Row lock Second thread waits its turn

Databases ship with these consistency tools for exactly this case.
