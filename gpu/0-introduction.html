Why concurrency

A modern CPU has many cores, and each core runs several threads. Serial code uses one — the rest sit idle.

Consumer CPU Count
Cores 4 – 8
Threads per core 1 – 4

A thread is an independent sequence of steps. Think of train tracks: apart, trains run at full speed; merged, each is limited by the train ahead. Independent work is free speed; shared work is where problems begin.

The OS scheduler decides which thread runs on which core. When a thread waits on data (a cache miss), the scheduler parks it and runs another. Switching costs time too, so constant switching is its own waste.

Memory is layered. Closer is faster.

Level Speed
Registers / L1 fastest, closest to core
L2 / L3 cache shared between cores
RAM → Disk slowest, farthest
