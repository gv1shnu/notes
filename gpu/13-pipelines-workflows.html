Concurrency patterns

13. Pipelines and workflows.

Both are directed graphs of steps. A pipeline is the straight-line case; a workflow can branch and rejoin.

Pipeline Workflow
Shape each step → one next step fan-out / fan-in, may cycle
Example step1 → step2 → step3 split work, run in parallel, merge

Divide-and-conquer and map-reduce can both be built as workflows.
