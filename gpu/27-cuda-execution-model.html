CUDA programming

Difficulty: Hard

27. Execution model.

The CPU (host) and GPU (device) have separate memory. Every kernel run follows the same five-step dance.

Step Action
1 allocate host + device memory
2 fill the input arrays on the host
3 copy host → device
4 launch the kernel
5 copy device → host, then free memory

The kernel runs once per thread. Threads are grouped into blocks, blocks into a grid.

Unit Note
Thread one kernel invocation
Block group of threads; size should be a multiple of 32 (a warp)
Grid all the blocks; can be 3D to match 3D data (video, maps)
In code 
C++

float *h_x, *d_x;
h_x = (float*)malloc(N * sizeof(float));      // host memory
cudaMalloc(&d_x, N * sizeof(float));         // device memory

cudaMemcpy(d_x, h_x, N*sizeof(float), cudaMemcpyHostToDevice);
add<<<blocks, threads>>>(N, d_x);             // launch kernel
cudaMemcpy(h_x, d_x, N*sizeof(float), cudaMemcpyDeviceToHost);

cudaFree(d_x); free(h_x);                     // clean up

The kernel runs once per thread (the dots). Threads are packed into blocks, and blocks tile the grid. Blocks are sized in multiples of 32 — one warp — because the hardware schedules 32 threads in lockstep.
