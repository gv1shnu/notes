Concurrency patterns

14. Recursion.

A function splits the data (often head / tail), works on the current piece, and calls itself on the rest. Needs a base case so it ends.

Good Bad
almost any problem can be expressed this way large data across distributed CPUs
natural in functional languages (Lisp, Clojure) GPUs perform poorly with it

That last point matters later: when we reach CUDA, recursion is the wrong tool.
