26. Runtime vs driver API.

The two ways your app reaches the GPU. Pick by how much low-level control you need.

Runtime API Driver API
Level high — an abstraction over the driver low — direct control
Setup automatic (modules, context) manual: init, device, context, module
GPU code embed inside host code compiled separately, loaded as cubin/fatbin
Written in C++ even assembly, or any language that links cubin
Use when most cases (course default) you truly need the control

Rule of thumb: start with the runtime API; reach for the driver API only when the extra power is worth the extra work.
