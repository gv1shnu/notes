30. Best practices.

Structure first.

Project Layout
Simple flat — all code in the project folder
Complex split into src/, include/, cmake/
Always a README with how to build and run

Then performance.

Lever Idea
Strong scaling (Amdahl) parallelize the easily-parallel parts first (e.g. unroll a loop across threads)
Weak scaling more threads ≠ linear speed-up; data won't always map 1:1 to threads
Branch divergence an if inside a warp makes the other threads wait → ~2× time
Memory helps, but won't rescue a bad algorithm; registers are limited

Fix divergence by pre-sorting data so a warp's threads take the same branch (e.g. group even/odd up front). Profile to find bottlenecks, test throughout, and print performance metrics as you run.
