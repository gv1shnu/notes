Pitfalls of shared state

5. Over- and under-utilization.

Using too many or too few threads. Both waste the machine.

Problem Cost
Too many threads idle, waiting for work context-switching cost > the speed-up
Too few threads always busy leaks compound, CPU spikes, more cache misses

Solution
Mechanism Effect
Scale thread count Match threads to the data volume or CPU utilization
