Pitfalls of shared state

4. Livelock.

Like deadlock, but the threads are busy — constantly reacting to each other, never actually progressing.

Two people in a hallway step aside to let the other pass, both the same way, again and again. A retry loop does the same: test, fail, sleep, retry, forever.

Deadlock Livelock
Threads frozen busy
Progress none none
Risk stuck stack/buffer overflow, infinite loop

Solution
Mechanism Effect
Random backoff Threads retry after different waits → break the symmetry

Misusing a language's async features can also cause it — be careful.
