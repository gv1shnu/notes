CUDA programming

28. Keywords.

Three groups. First, where code runs — part of the function signature, not the call.

Qualifier Runs on Called from
__host__ CPU CPU
__global__ GPU (a kernel) CPU
__device__ GPU GPU (called by other GPU code)

Second, the launch config — how the work is divided: 

kernel<<< blocksPerGrid, threadsPerBlock, sharedMem, stream >>>(args);

Third, where data lives:

Memory Scope
registers fastest, per-thread (limited)
__constant__ read-only, shared
__shared__ per-block; not guaranteed — may spill to global
__device__ device global memory
In code — a kernel: y = a*x + y, one thread per element. 
CUDA

__global__ void saxpy(int n, float a, float *x, float *y) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;   // this thread's index
    if (i < n) y[i] = a * x[i] + y[i];
}

// launch: 256 threads per block, enough blocks to cover n
saxpy<<<(n + 255) / 256, 256>>>(n, 2.0f, d_x, d_y);
