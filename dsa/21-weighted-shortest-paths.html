Weighted Shortest Paths

Now each edge carries a weight (time, cost, distance) and we minimize the sum along the path — not the edge count, so BFS no longer applies. Two engines do it, split by whether negative weights are allowed.

The shared primitive: edge relaxation

Keep dist[v] = best path length to v found so far (an upper bound, starting at ∞, with dist[s]=0). Relaxing edge (u, v) asks: is going through u cheaper? 

relax(u, v):
    if dist[v] > dist[u] + w(u, v):
        dist[v] = dist[u] + w(u, v)
        prev[v] = u

It's correct because of optimal substructure: every subpath of a shortest path is itself shortest. Both algorithms below are just different schedules of relaxations.

Dijkstra — non-negative weights

Greedy: repeatedly pick the unfinalized vertex with the smallest dist, finalize it, and relax its outgoing edges. Once picked, its distance is final — but only because no later (non-negative) edge can make a path cheaper. A single negative edge breaks that guarantee.

Priority queue Running time Best when
array O(|V|²) dense graphs (|E| ≈ |V|²)
binary heap O((|V| + |E|) log |V|) sparse graphs 
import heapq

def dijkstra(graph, s):             # graph: u -> [(v, w), ...], w >= 0
    dist = {u: float('inf') for u in graph}
    dist[s] = 0
    pq = [(0, s)]                   # (distance, node) min-heap
    while pq:
        d, u = heapq.heappop(pq)
        if d > dist[u]: continue    # stale entry, skip
        for v, w in graph[u]:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                heapq.heappush(pq, (dist[v], v))
    return dist

vector<vector<pair<int,int>>> graph;   // graph[u] = {(v, w), ...}, w >= 0

vector<long long> dijkstra(int n, int s) {
    const long long INF = 1e18;
    vector<long long> dist(n, INF);
    dist[s] = 0;
    priority_queue<pair<long long,int>,
        vector<pair<long long,int>>, greater<>> pq;  // min-heap
    pq.push({0, s});
    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (d > dist[u]) continue;      // stale entry, skip
        for (auto [v, w] : graph[u])
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
    }
    return dist;
}

Bellman-Ford — negative weights allowed

Drop the greedy order: just relax every edge, and repeat |V| − 1 times. Why |V|−1? A shortest path is simple, so it has at most |V|−1 edges; one full sweep extends every correct distance by one more edge.

Dijkstra Bellman-Ford
Negative edges no yes (no negative cycle)
Strategy greedy + priority queue relax all edges, |V|−1 passes
Time O((|V|+|E|) log|V|) O(|V| · |E|) 
def bellman_ford(n, edges, s):      # edges: list of (u, v, w)
    dist = [float('inf')] * n
    dist[s] = 0
    for _ in range(n - 1):          # |V|-1 relaxation passes
        for u, v, w in edges:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
    for u, v, w in edges:           # extra pass: any relax => neg cycle
        if dist[u] + w < dist[v]:
            return None             # negative cycle reachable
    return dist

struct Edge { int u, v, w; };

vector<long long> bellman_ford(int n, vector<Edge>& edges, int s, bool& neg) {
    const long long INF = 1e18;
    vector<long long> dist(n, INF);
    dist[s] = 0;
    for (int i = 0; i < n - 1; ++i)     // |V|-1 relaxation passes
        for (auto& e : edges)
            if (dist[e.u] < INF && dist[e.u] + e.w < dist[e.v])
                dist[e.v] = dist[e.u] + e.w;
    neg = false;
    for (auto& e : edges)               // extra pass detects neg cycle
        if (dist[e.u] < INF && dist[e.u] + e.w < dist[e.v]) neg = true;
    return dist;
}

Negative cycles & arbitrage

If the graph has a cycle of negative total weight, "shortest" is meaningless — loop it forever and the distance → −∞. Detect it with one extra (|V|-th) pass: if any edge still relaxes, a negative cycle is reachable. Trace it by following prev backwards from a vertex that relaxed on that pass.

Currency arbitrage is exactly this: take −log of each exchange rate, and a negative cycle = a loop of trades that multiplies your money. Infinite arbitrage reaches currency u iff u is reachable (run BFS from every node that relaxed on pass |V|) from such a cycle — a negative cycle existing isn't enough; u must be downstream of one.

Language notes

  • heapq has no decrease-key — push a fresh (dist, node) and skip stale pops with the if d > dist[u]: continue guard.
  • Tuples compare lexicographically, so (dist, node) naturally orders by distance.
  • float('inf') absorbs + w safely, so Bellman-Ford needs no overflow guard.
  • priority_queue is a max-heap — add greater<> for a min-heap. Same lazy-deletion trick (no decrease-key).
  • Use a long long INF = 1e18 and guard dist[u] < INF before adding, or sums overflow.
  • Structured bindings auto [v, w] unpack the adjacency pairs cleanly.
