Linked Lists

Nodes chained by pointers. Each node holds a key and a next pointer; a head points to the first. No contiguous memory — so no O(1) random access, but cheap insert/remove at the ends. 

class Node:
    def __init__(self, key):
        self.key, self.next = key, None

class LinkedList:
    def __init__(self):
        self.head = self.tail = None

    def push_front(self, key):        # O(1)
        n = Node(key); n.next = self.head; self.head = n
        if self.tail is None: self.tail = n

    def push_back(self, key):         # O(1) thanks to tail
        n = Node(key)
        if self.tail: self.tail.next = n
        else:         self.head = n
        self.tail = n

    def pop_front(self):              # O(1)
        if not self.head: return None
        n = self.head; self.head = n.next
        if self.head is None: self.tail = None
        return n.key

struct Node {
    int key; Node* next = nullptr;
    Node(int k) : key(k) {}
};
struct LinkedList {
    Node *head = nullptr, *tail = nullptr;

    void push_front(int key) {        // O(1)
        Node* n = new Node(key); n->next = head; head = n;
        if (!tail) tail = n;
    }
    void push_back(int key) {         // O(1) thanks to tail
        Node* n = new Node(key);
        if (tail) tail->next = n; else head = n;
        tail = n;
    }
    int pop_front() {                 // O(1)
        Node* n = head; head = head->next;
        if (!head) tail = nullptr;
        int k = n->key; delete n; return k;
    }
};

Singly-linked

Only forward pointers. A tail pointer makes appending cheap, but you still can't cheaply reach the second-to-last node (no back-pointer), so popping the back stays O(n).

Operation No tail With tail
PushFront / TopFront / PopFront O(1) O(1)
PushBack / TopBack O(n) O(1)
PopBack O(n) O(n)
Find / Erase / AddBefore O(n) O(n)
AddAfter(node) O(1) O(1)

Doubly-linked

Each node also has a prev pointer. Now you can walk backwards, so PopBack and AddBefore become O(1) — at the cost of one extra pointer per node.

Array vs linked list

Array Linked list
Random access A[i] O(1) O(n)
Insert/remove at front O(n) O(1)
Size fixed / amortized resize grows freely
Overhead may over-allocate a pointer per node

Language notes

  • No built-in singly-linked list. collections.deque is a doubly-linked list of fixed-size blocks: O(1) append/appendleft/pop/popleft.
  • But deque indexing in the middle is O(n) — it's for ends, not random access.
  • A plain list is not a linked list (it's a dynamic array); you'd only hand-roll nodes to study the structure.
  • std::list = doubly-linked: O(1) insert/erase given an iterator, no random access. std::forward_list = singly-linked, one pointer less per node.
  • Splicing (list::splice) moves nodes between lists in O(1) without copying.
  • A node per element kills cache locality — in practice a vector often beats a list even for mid-inserts. Measure before reaching for it.
