Linked Lists

Nodes chained by pointers. Each node holds a key and a next pointer; a head points to the first. No contiguous memory — so no O(1) random access, but cheap insert/remove at the ends.

Singly-linked

Only forward pointers. A tail pointer makes appending cheap, but you still can't cheaply reach the second-to-last node (no back-pointer), so popping the back stays O(n).

Operation No tail With tail
PushFront / TopFront / PopFront O(1) O(1)
PushBack / TopBack O(n) O(1)
PopBack O(n) O(n)
Find / Erase / AddBefore O(n) O(n)
AddAfter(node) O(1) O(1)

Doubly-linked

Each node also has a prev pointer. Now you can walk backwards, so PopBack and AddBefore become O(1) — at the cost of one extra pointer per node.

Array vs linked list

Array Linked list
Random access A[i] O(1) O(n)
Insert/remove at front O(n) O(1)
Size fixed / amortized resize grows freely
Overhead may over-allocate a pointer per node
