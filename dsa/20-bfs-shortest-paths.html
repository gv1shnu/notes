BFS & Shortest Paths (unweighted)

The length of a path is its number of edges; the distance d(s, v) is the length of the shortest path (∞ if v is unreachable). Breadth-first search computes d(s, ·) to every vertex at once — e.g. fewest flight transfers from one city to all others.

Distance layers

Group vertices by distance from s: layer 0 = {s}, layer 1 = s's neighbors, and so on. The structure constrains where edges can go — an edge that jumped two layers forward would be a shortcut, contradicting the layering:

Edge from layer i can reach… Undirected Directed
same or adjacent layer (i, i±1) yes yes
any earlier layer (≤ i) — yes
two+ layers forward (≥ i+2) impossible impossible

BFS

Visit layer by layer using a queue (FIFO) — that's the one change from DFS, which rides a stack. Dequeue a vertex, and any neighbor still at distance ∞ is newly discovered: set its distance and enqueue it. 

from collections import deque

def bfs(graph, s):                  # graph: dict v -> neighbors
    dist = {v: float('inf') for v in graph}
    prev = {v: None for v in graph}
    dist[s] = 0
    q = deque([s])
    while q:
        u = q.popleft()
        for v in graph[u]:
            if dist[v] == float('inf'):   # newly discovered
                dist[v] = dist[u] + 1
                prev[v] = u
                q.append(v)
    return dist, prev

def reconstruct(prev, s, t):        # shortest path s -> t
    path, u = [], t
    while u is not None:
        path.append(u); u = prev[u]
    path.reverse()
    return path if path and path[0] == s else []  # [] if unreachable

vector<vector<int>> graph;          // graph[u] = neighbors

pair<vector<int>, vector<int>> bfs(int n, int s) {
    vector<int> dist(n, -1), prev(n, -1);   // -1 = infinity / none
    dist[s] = 0;
    queue<int> q; q.push(s);
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int v : graph[u])
            if (dist[v] == -1) {            // newly discovered
                dist[v] = dist[u] + 1;
                prev[v] = u;
                q.push(v);
            }
    }
    return {dist, prev};
}
vector<int> reconstruct(const vector<int>& prev, int s, int t) {
    vector<int> path;
    for (int u = t; u != -1; u = prev[u]) path.push_back(u);
    reverse(path.begin(), path.end());
    if (path.empty() || path.front() != s) return {};  // unreachable
    return path;
}

Each vertex is enqueued at most once (its distance leaves ∞ the moment it's found and never re-enters the queue), so every edge is scanned at most twice — O(|V| + |E|), linear.

Shortest-path tree

Distances alone don't tell you the route. Fix that with one extra array: when v is discovered while processing u, record prev[v] = u. Following prev from any vertex back to s — each hop drops the distance by exactly 1 — traces a shortest path; reverse it to read it forward. These prev pointers form a tree (no cycles: distances strictly decrease along every hop, so a loop is impossible) rooted at s.

BFS DFS
Frontier queue (FIFO) stack / recursion
Finds shortest paths? yes (unweighted) no
Order nearest first, layer by layer deepest first

BFS gives the right answer only when every edge "costs" 1. The moment edges carry different weights, layer-by-layer breaks — that's Dijkstra and Bellman-Ford.

Language notes

  • collections.deque is the queue — append + popleft O(1). A list with pop(0) turns BFS into O(V·n).
  • For grid problems, encode a cell (r, c) as a tuple key or a flat r*W + c integer.
  • std::queue<int> over a deque; at scale a preallocated vector + head index is faster.
  • Use a -1 sentinel for unvisited dist/prev — one array does double duty as the "visited" flag.
