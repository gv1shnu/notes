Queues

A FIFO collection — First In, First Out. Like a line at a checkout: longest-waiting is served next.

Operation Does
Enqueue(key) add to the back
Dequeue() remove + return the front (oldest)
Empty() any elements?

All O(1). Two implementations:

Built on How
Linked list (head + tail) Enqueue = PushBack, Dequeue = PopFront; grows freely
Circular array read + write indices that wrap around; fixed size

The circular array keeps a read and write index; both wrap to 0 at the end. One slot is always left unused so read == write unambiguously means "empty" (not "full"). That's what keeps both Enqueue and Dequeue O(1) instead of an O(n) shift. 

class CircularQueue:
    def __init__(self, cap):
        self.data = [None] * (cap + 1)   # one slot left empty
        self.read = self.write = 0

    def enqueue(self, x):                # O(1)
        self.data[self.write] = x
        self.write = (self.write + 1) % len(self.data)

    def dequeue(self):                   # O(1)
        x = self.data[self.read]
        self.read = (self.read + 1) % len(self.data)
        return x

    def empty(self):
        return self.read == self.write

struct CircularQueue {
    vector<int> data; int read = 0, write = 0;
    CircularQueue(int cap) : data(cap + 1) {}   // one slot left empty

    void enqueue(int x) {                // O(1)
        data[write] = x;
        write = (write + 1) % data.size();
    }
    int dequeue() {                      // O(1)
        int x = data[read];
        read = (read + 1) % data.size();
        return x;
    }
    bool empty() { return read == write; }
};

Stacks are LIFO, queues are FIFO — that one difference is why a queue is right for servers, scheduling, and breadth-first search, while a stack fits undo and depth-first search.

Watch it run

A circular buffer of 6 slots (capacity 5 — one slot is always kept empty). W is where the next Enqueue writes, R is the front the next Dequeue reads; both wrap past the end. Watch read chase write around the ring. Reset generates a new op sequence.

Language notes

  • collections.deque is the queue: append + popleft both O(1). Never use a list as a queue — pop(0) is O(n).
  • deque(maxlen=k) gives a fixed-size ring buffer that auto-drops the oldest item.
  • queue.Queue is a thread-safe wrapper (locks) for producer/consumer pipelines — correctness over speed.
  • std::queue<T> is an adapter over std::deque; push/pop/front/back are O(1).
  • std::deque itself already gives O(1) at both ends (segmented storage) — use it directly when you need indexing too.
  • For BFS over a fixed graph, a preallocated vector + head/tail indices is often faster than queue.
