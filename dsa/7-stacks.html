Stacks

A LIFO collection — Last In, First Out. Like a stack of books: you only touch the top.

Operation Does
Push(key) add to the top
Top() peek at the most recent
Pop() remove + return the most recent
Empty() any elements?

Every operation is O(1), on either implementation:

Built on Trade-off
Array (+ count) fixed max size; may waste allocated space
Linked list (Push = PushFront) unbounded; one pointer of overhead per element

Classic use: balanced brackets

Check that ([])() is balanced. Scan left to right: push every opening bracket; on a closing bracket, pop and check it matches. At the end the stack must be empty. 

def is_balanced(s):
    pairs = {')': '(', ']': '[', '}': '{'}
    stack = []
    for ch in s:
        if ch in '([{':
            stack.append(ch)          # opening -> push
        elif ch in pairs:
            if not stack or stack.pop() != pairs[ch]:
                return False      # mismatch / nothing to match
    return not stack                  # leftover opens = unbalanced

bool is_balanced(const string& s) {
    auto open_of = [](char c){ return c==')'?'(' : c==']'?'[' : '{'; };
    stack<char> st;
    for (char ch : s) {
        if (ch=='('||ch=='['||ch=='{') st.push(ch);  // push
        else if (ch==')'||ch==']'||ch=='}') {
            if (st.empty() || st.top() != open_of(ch)) return false;
            st.pop();
        }
    }
    return st.empty();                // leftover opens = unbalanced
}

The stack naturally remembers the still-open brackets in the exact order they must close — that's what LIFO is for. (Also: function call stacks, expression parsing, undo.)

Language notes

  • Use a plain list as a stack: append = push, pop() = pop, both O(1) amortized; list[-1] peeks.
  • Never use insert(0, x) / pop(0) for a stack — those are O(n).
  • There's no dedicated stack type; the list idiom is what everyone uses.
  • std::stack<T> is an adapter over std::deque by default (can wrap vector/list); push/pop/top are O(1).
  • pop() returns void — read top() first, then pop().
  • For raw speed, a vector with push_back/back/pop_back is the same thing with less indirection.
