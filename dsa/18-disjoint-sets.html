Disjoint Sets (Union-Find)

Track a collection of non-overlapping groups, with two questions: "are these two in the same group?" and "merge these two groups." Three operations:

Operation Does
MakeSet(x) new group containing just x
Find(x) return x's group id (its root)
Union(x, y) merge the two groups

Forest of trees

Represent each group as a tree; each node points to a parent, the root is the group id. Find walks to the root; Union hangs one root under the other. Done carelessly, trees become long chains and everything is O(n). Two heuristics fix that.

Heuristic Idea
Union by rank always hang the shorter tree under the taller one → height stays O(log n)
Path compression during Find, re-attach every node on the path directly to the root 
class DisjointSets:
    def __init__(self, n):
        self.parent = list(range(n))   # each its own root
        self.rank = [0] * n

    def find(self, x):                 # path compression
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x, y):             # union by rank
        rx, ry = self.find(x), self.find(y)
        if rx == ry: return
        if self.rank[rx] < self.rank[ry]: rx, ry = ry, rx
        self.parent[ry] = rx           # shorter under taller
        if self.rank[rx] == self.rank[ry]: self.rank[rx] += 1

struct DisjointSets {
    vector<int> parent, rnk;
    DisjointSets(int n) : parent(n), rnk(n, 0) {
        for (int i = 0; i < n; ++i) parent[i] = i;   // each its own root
    }
    int find(int x) {                  // path compression
        if (parent[x] != x) parent[x] = find(parent[x]);
        return parent[x];
    }
    void unite(int x, int y) {         // union by rank
        int rx = find(x), ry = find(y);
        if (rx == ry) return;
        if (rnk[rx] < rnk[ry]) swap(rx, ry);
        parent[ry] = rx;               // shorter under taller
        if (rnk[rx] == rnk[ry]) rnk[rx]++;
    }
};

Near-constant time

With union by rank alone, every operation is O(log n). Add path compression and the amortized cost drops to O(log* n) — the iterated logarithm, which is ≤ 5 for any input that fits in the universe. So in practice, effectively constant per operation.

(Used for: connected components, Kruskal's MST, "are these networked together?")

Language notes

  • No stdlib union-find — hand-roll over a list (it's tiny). This is the backbone of Kruskal's MST.
  • Recursive find can hit the recursion limit on a long chain before compression flattens it — an iterative two-pass find is safer for large n.
  • No stdlib structure either; a pair of vector<int> (parent + rank) is the standard pattern. (boost::disjoint_sets exists if you want it.)
  • Prefer iterative find at very large n to avoid deep recursion; some use union by size instead of rank — same near-constant bound.
