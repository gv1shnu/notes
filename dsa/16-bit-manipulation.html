Bit Manipulation

Work on a number's binary representation directly. It's how computers already think (everything is 1s and 0s), so bit ops are fast, branch-free, and perfect for flags, sets, and squeezing constants out of hot loops.

The operators

Op Does
& AND 1 only where both bits are 1 (masking)
| OR 1 where either bit is 1 (setting)
^ XOR 1 where bits differ (toggling)
~ NOT flip every bit
<< k shift left = multiply by 2k
>> k shift right = floor-divide by 2k

The everyday operations on bit i

x & 1            # 1 if x is odd (parity)
(x >> i) & 1     # read bit i
x | (1 << i)     # set bit i
x & ~(1 << i)   # clear bit i
x ^ (1 << i)    # toggle bit i

The tricks worth memorizing

Goal Trick Why
Is it a power of two? x > 0 and x & (x - 1) == 0 a power of two has exactly one set bit
Drop the lowest set bit x & (x - 1) loop it to count set bits (Kernighan)
Isolate the lowest set bit x & -x the basis of Fenwick / BIT trees
Swap without a temp a ^= b; b ^= a; a ^= b XOR cancels itself

And the headline use in algorithms: a single integer is a set — bit i = "element i present." That bitmask turns subset enumeration and DP over subsets into plain integer loops over 0 .. 2n-1.

Language notes

  • int is arbitrary precision — no fixed width, no overflow, and ~x behaves as two's complement -(x+1) (infinite sign bits).
  • Helpers: bin(x), x.bit_length(), and x.bit_count() (3.10+) for popcount.
  • Fixed width — mind overflow, and shifting by ≥ the type's width (or shifting signed negatives) is undefined behaviour; use unsigned for masks.
  • C++20 <bit>: std::popcount, countl_zero, has_single_bit; or the GCC builtins __builtin_popcount / __builtin_ctz.
