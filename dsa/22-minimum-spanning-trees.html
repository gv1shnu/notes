Minimum Spanning Trees

Given a connected, undirected, weighted graph, a minimum spanning tree (MST) is the cheapest set of edges that connects all vertices — e.g. wiring up a network with the least total cable. It spans every vertex, has no cycle, and minimizes total weight. Both standard algorithms are greedy, and both are correct for the same reason.

The cut property (why greedy is safe)

Split the vertices into any two groups S and V−S such that no edge chosen so far crosses the split. Then the lightest edge crossing that cut is safe — it belongs to some MST. (Proof: swapping it for any heavier crossing edge of an MST keeps the tree connected and no heavier.) Add safe edges repeatedly and you build an MST. Kruskal and Prim just choose the cut differently.

Kruskal — cheapest edge, globally

Sort all edges ascending; add the next lightest edge that doesn't form a cycle. The partial solution is a forest that gradually merges into one tree. "Same component?" is exactly the disjoint-sets (union-find) query: 

def kruskal(n, edges):              # edges: list of (w, u, v)
    ds = DisjointSets(n)            # from the disjoint-sets page
    edges.sort()                    # ascending by weight (w first)
    mst, total = [], 0
    for w, u, v in edges:
        if ds.find(u) != ds.find(v):   # different trees -> no cycle
            ds.union(u, v)
            mst.append((u, v)); total += w
    return mst, total

struct Edge { int w, u, v; };       // weight first for easy sort

long long kruskal(int n, vector<Edge>& edges) {
    DisjointSets ds(n);             // from the disjoint-sets page
    sort(edges.begin(), edges.end(),
         [](auto& a, auto& b){ return a.w < b.w; });
    long long total = 0;
    for (auto& e : edges)
        if (ds.find(e.u) != ds.find(e.v)) {  // no cycle
            ds.unite(e.u, e.v);
            total += e.w;
        }
    return total;
}

Dominated by the sort: O(|E| log |V|). (If edges arrive pre-sorted, the union-find passes run in near-linear O(|E| log* |V|).)

Prim — grow one tree

Pick any root and grow a single tree, each step attaching the new vertex reachable by the lightest edge. A priority queue keyed by each outside vertex's cheapest connecting edge gives the next vertex — structurally Dijkstra, but the key is the single edge weight, not the accumulated path length. 

import heapq

def prim(graph, s=0):               # graph: u -> [(v, w), ...]
    visited = set()
    pq = [(0, s)]                   # (edge weight, node)
    total = 0
    while pq:
        w, u = heapq.heappop(pq)
        if u in visited: continue   # already in the tree
        visited.add(u); total += w
        for v, wt in graph[u]:
            if v not in visited:
                heapq.heappush(pq, (wt, v))
    return total

vector<vector<pair<int,int>>> graph;   // graph[u] = {(v, w), ...}

long long prim(int n, int s = 0) {
    vector<bool> visited(n, false);
    priority_queue<pair<int,int>,
        vector<pair<int,int>>, greater<>> pq;
    pq.push({0, s});                // (edge weight, node)
    long long total = 0;
    while (!pq.empty()) {
        auto [w, u] = pq.top(); pq.pop();
        if (visited[u]) continue;   // already in the tree
        visited[u] = true; total += w;
        for (auto [v, wt] : graph[u])
            if (!visited[v]) pq.push({wt, v});
    }
    return total;
}
Kruskal Prim
Grows a forest that merges one tree from a root
Picks globally lightest safe edge lightest edge leaving the tree
Helper structure disjoint sets priority queue
Time O(|E| log |V|) O(|E| log |V|) heap · O(|V|²) array

Same MST, same cost — both ride the cut property. Kruskal leans on union-find, Prim on a heap; pick by which structure is already at hand and whether the graph is sparse (Kruskal / Prim-heap) or dense (Prim-array).

Language notes

  • Kruskal reuses the union-find; storing edges as (w, u, v) lets the default sort() order them by weight.
  • Prim reuses heapq with the same lazy-deletion trick as Dijkstra (skip pops for nodes already in the tree) — no decrease-key needed.
  • Kruskal = sort edges + the DisjointSets struct; Prim mirrors Dijkstra with a greater<> min-heap.
  • Choose by density: Kruskal / heap-Prim for sparse graphs, an O(V²) array-Prim for dense ones.
