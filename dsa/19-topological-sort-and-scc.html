Topological Sort & Strongly Connected Components

Difficulty: Hard

Two classic uses of DFS on directed graphs. DFS works the same as on undirected graphs — you just follow each edge only in its arrow's direction.

DAGs

A directed acyclic graph is a directed graph with no cycle. They model dependencies: draw A→B for "A must come before B" (morning routine, build steps, course prerequisites). Two named vertex roles:

Term Meaning
Source no incoming edges
Sink no outgoing edges

Topological sort

Order the vertices on a line so every edge points forward (left→right). This is possible if and only if the graph is a DAG — a cycle (chicken↔egg) has no valid first element, so it can't be ordered.

Approach Cost
Naive repeatedly find a sink, append it, delete it — re-walks the whole path each time O(|V|²)
DFS run DFS, output vertices in decreasing post number (reverse post-order) O(|V| + |E|)

Why reverse post-order works: for any edge u→v in a DAG, post(u) > post(v) always — so listing vertices from highest post to lowest puts every edge's tail before its head. A sink is just the vertex that finishes first (smallest post), and it lands last. 

def topological_sort(graph):       # graph: dict v -> successors
    visited, order = set(), []
    def dfs(v):
        visited.add(v)
        for w in graph[v]:
            if w not in visited: dfs(w)
        order.append(v)            # post-order: v is finished
    for v in graph:
        if v not in visited: dfs(v)
    return order[::-1]              # reverse post-order

vector<vector<int>> graph;          // graph[v] = successors
vector<int> order; vector<bool> visited;

void dfs(int v) {
    visited[v] = true;
    for (int w : graph[v]) if (!visited[w]) dfs(w);
    order.push_back(v);             // post-order: v is finished
}
vector<int> topological_sort(int n) {
    visited.assign(n, false); order.clear();
    for (int v = 0; v < n; ++v) if (!visited[v]) dfs(v);
    reverse(order.begin(), order.end());   // reverse post-order
    return order;
}

Strongly connected components

In a directed graph, two vertices are connected only if you can go from each to the other and back. That relation partitions the graph into SCCs: inside one, everything reaches everything; leave one and you can't return.

Collapse each SCC to a single node and you get the metagraph (condensation) — which is always a DAG. (If it had a cycle, those SCCs would all reach each other and merge into one.) That DAG structure is what the algorithm exploits.

Computing SCCs — two DFS passes (Kosaraju)

Key fact: the vertex with the largest post number in the reverse graph GR (all edges flipped) lies in a sink SCC of G — and exploring from a sink-SCC vertex finds exactly that one SCC (nothing leaks out). So:

Step Do
1 DFS on GR, record post numbers
2 process vertices of G in decreasing post number; each unvisited one you explore peels off exactly one SCC

Two linear DFS passes — GR then G — so the whole thing is O(|V| + |E|), down from the naive O(|V|² + |V||E|) of exploring from every vertex. 

def kosaraju(graph, n):            # returns comp[v] = SCC id
    rev = [[] for _ in range(n)]
    for v in range(n):
        for w in graph[v]: rev[w].append(v)   # reverse graph

    visited, order = set(), []
    def dfs1(v):                   # pass 1: finish order on G^R
        visited.add(v)
        for w in rev[v]:
            if w not in visited: dfs1(w)
        order.append(v)
    for v in range(n):
        if v not in visited: dfs1(v)

    comp = [-1] * n
    def dfs2(v, label):            # pass 2: explore G
        comp[v] = label
        for w in graph[v]:
            if comp[w] == -1: dfs2(w, label)
    label = 0
    for v in reversed(order):       # decreasing finish time
        if comp[v] == -1:
            dfs2(v, label); label += 1
    return comp

vector<vector<int>> graph, rev;
vector<int> order, comp;

void dfs1(int v, vector<bool>& vis) {     // pass 1 on G^R
    vis[v] = true;
    for (int w : rev[v]) if (!vis[w]) dfs1(w, vis);
    order.push_back(v);
}
void dfs2(int v, int label) {           // pass 2 on G
    comp[v] = label;
    for (int w : graph[v]) if (comp[w] == -1) dfs2(w, label);
}
void kosaraju(int n) {
    rev.assign(n, {});
    for (int v = 0; v < n; ++v)
        for (int w : graph[v]) rev[w].push_back(v);  // reverse graph
    vector<bool> vis(n, false); order.clear();
    for (int v = 0; v < n; ++v) if (!vis[v]) dfs1(v, vis);
    comp.assign(n, -1);
    int label = 0;
    for (int i = n - 1; i >= 0; --i)      // decreasing finish time
        if (comp[order[i]] == -1) dfs2(order[i], label++);
}

Language notes

  • graphlib.TopologicalSorter (3.9+) gives a topo order straight from a {node: {deps}} dict — no hand-rolled DFS needed.
  • Recursive DFS on a deep graph hits the recursion limit; flip to an explicit stack (the iterative DFS must push post-order carefully).
  • No stdlib topo/SCC — hand-roll. Tarjan's SCC does it in a single DFS pass (low-link values) vs Kosaraju's two — same O(|V|+|E|), often preferred.
  • For topo order you can also use Kahn's algorithm (repeatedly remove in-degree-0 nodes via a queue) — no recursion at all.
