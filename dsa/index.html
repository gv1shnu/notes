Data Structures and Algorithms

Two halves that only make sense together. Data structures hold the data; algorithms act on it. Some algorithms demand a particular structure — binary search needs a sorted array, heapsort needs a heap — while others (the paradigms) run on almost anything.

Foundations

  • Introduction — time & space complexity, Big-O

Data Structures

The containers — how data is held, and what each access costs.

  • Arrays — contiguous, O(1) random access; dynamic arrays
  • Linked Lists — cheap ends, no random access
  • Stacks — LIFO
  • Queues — FIFO
  • Priority Queues & Heaps — max/min out first
  • Disjoint Sets (Union-Find) — merge & query groups
  • Hash Tables — O(1) average key→value
  • Trees — BST, AVL, splay
  • Graphs — representation, DFS, connectivity

Graph Algorithms

The methods that run on a graph — built on DFS, BFS, greedy, and priority queues.

  • Topological Sort & SCC — DAGs, ordering, Kosaraju
  • BFS & Shortest Paths — unweighted distances, layers
  • Weighted Shortest Paths — Dijkstra, Bellman-Ford
  • Minimum Spanning Trees — cut property, Kruskal, Prim
  • Advanced Shortest Paths — bidirectional, A*, contraction hierarchies

Algorithms

The methods — and the structure each one leans on.

Algorithm Runs on
Searching an array — binary search needs it sorted
Sorting an array — heapsort uses a heap; counting/radix need small-integer keys
Recursion (soon) the call stack
Divide & Conquer any
Greedy any — often paired with sorting or a heap
Dynamic Programming an array / table to memoize
Backtracking (soon) any — via recursion (the stack)
Bit Manipulation (soon) integers
