Priority Queues & Heaps

A priority queue serves the highest-priority item next (not the oldest, like a queue). The efficient implementation is a binary heap.

Operation Naive (array) Heap
Insert O(1) O(log n)
ExtractMax O(n) O(log n)
GetMax O(n) O(1)

The heap property

A binary max-heap is a binary tree where every parent ≥ its children. So the max sits at the root. Two operations fix violations, each costing O(tree height):

Op Does
SiftUp after Insert: swap a too-big node upward until parent ≥ it
SiftDown after ExtractMax: swap a too-small node down past its larger child

Keep it shallow + array-backed

Force a complete binary tree (all levels full, last level filled left-to-right) so height is always O(log n). The payoff: no pointers needed — store it in a plain array, indices computed on the fly.

Node i Index
parent ⌊i / 2⌋
left child 2i
right child 2i + 1

Insert at the last position then SiftUp; ExtractMax swaps root with the last leaf then SiftDown — both keep the tree complete. 

class MaxHeap:
    def __init__(self):
        self.h = [None]                # index 0 unused (1-indexed)

    def sift_up(self, i):
        while i > 1 and self.h[i] > self.h[i // 2]:   # bigger than parent
            self.h[i], self.h[i // 2] = self.h[i // 2], self.h[i]
            i //= 2

    def sift_down(self, i):
        n = len(self.h) - 1
        while 2 * i <= n:
            j = 2 * i                          # larger child
            if j + 1 <= n and self.h[j + 1] > self.h[j]: j += 1
            if self.h[i] >= self.h[j]: break
            self.h[i], self.h[j] = self.h[j], self.h[i]
            i = j

    def insert(self, x):                  # O(log n)
        self.h.append(x); self.sift_up(len(self.h) - 1)

    def extract_max(self):                # O(log n)
        n = len(self.h) - 1
        self.h[1], self.h[n] = self.h[n], self.h[1]
        top = self.h.pop()
        if len(self.h) > 1: self.sift_down(1)
        return top

struct MaxHeap {
    vector<int> h{0};                   // index 0 unused (1-indexed)

    void sift_up(int i) {
        while (i > 1 && h[i] > h[i / 2]) {      // bigger than parent
            swap(h[i], h[i / 2]); i /= 2;
        }
    }
    void sift_down(int i) {
        int n = h.size() - 1;
        while (2 * i <= n) {
            int j = 2 * i;                      // larger child
            if (j + 1 <= n && h[j + 1] > h[j]) ++j;
            if (h[i] >= h[j]) break;
            swap(h[i], h[j]); i = j;
        }
    }
    void insert(int x) {                  // O(log n)
        h.push_back(x); sift_up(h.size() - 1);
    }
    int extract_max() {                  // O(log n)
        int n = h.size() - 1;
        swap(h[1], h[n]);
        int top = h.back(); h.pop_back();
        if (h.size() > 1) sift_down(1);
        return top;
    }
};

BuildHeap & HeapSort

SiftDown every internal node bottom-up to turn an array into a heap. Naively that looks like O(n log n), but most nodes are near the leaves — the sum works out to O(n). HeapSort = build the heap (O(n)), then ExtractMax n times (O(n log n)) → O(n log n), in place. Also great for "top k": build-heap + k extracts = O(n) when k ≤ n/log n.

Language notes

  • heapq is a min-heap over a plain list: heappush/heappop O(log n), heapify O(n).
  • Want a max-heap? Push negated values (or (-priority, item) tuples).
  • heapq.nlargest/nsmallest(k, it) for top-k; heappushpop / heapreplace fuse push+pop in one O(log n) step.
  • std::priority_queue is a max-heap by default (over a vector): push/pop/top.
  • Min-heap: priority_queue<int, vector<int>, greater<int>>.
  • Lower-level: std::make_heap/push_heap/pop_heap work on a raw range; nth_element does O(n) selection without a heap.
