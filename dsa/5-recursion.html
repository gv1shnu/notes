Recursion

A function that solves a problem by calling itself on a smaller version of the same problem. It fits when a problem is self-similar — a factorial, a tree, a divide-and-conquer split. Every recursion rides the call stack: each call is a frame pushed on, returned frames pop off.

Two parts, always

Part Job
Base case the smallest input you can answer directly — stops the recursion
Recursive case do a bit of work, then call yourself on a strictly smaller input

Miss the base case (or never shrink toward it) and the stack grows forever — a stack overflow. The two rules: have a base case, and always move toward it. 

def factorial(n):
    if n <= 1:           # base case
        return 1
    return n * factorial(n - 1)   # recursive case

long long factorial(int n) {
    if (n <= 1) return 1;          // base case
    return n * factorial(n - 1);   // recursive case
}

Recursion vs iteration

Anything recursive can be rewritten with a loop and an explicit stack — recursion just lets the language manage that stack for you. The trade is clarity vs control:

Recursion Iteration
Stack the call stack (hidden) an explicit stack / none
Reads as the problem's definition bookkeeping
Risk stack overflow on deep input safe at any depth

The optimization: stop redoing work

Naive recursion on overlapping subproblems explodes — fib(n) recomputes the same values exponentially (see the intro). Cache each answer once (memoization) and it collapses to linear; that move is dynamic programming. Recursion is also the spine of divide & conquer, tree traversal, and backtracking.

Language notes

  • Default recursion limit is ~1000 frames; deeper hits RecursionError. Raise it with sys.setrecursionlimit or convert to a loop.
  • CPython does no tail-call optimization — a "tail recursive" function still grows the stack.
  • Memoize for free with @functools.lru_cache(None).
  • Recursion uses the real call stack (~1 MB) — overflow segfaults silently rather than throwing.
  • Compilers may optimize tail calls (-O2), but it isn't guaranteed — don't rely on it for deep recursion.
  • For very deep work, hand-roll an explicit std::stack.
