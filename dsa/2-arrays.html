Arrays

A contiguous block of memory, split into equal-size elements indexed by contiguous integers. Those three facts buy the superpower: O(1) random access — the address of element i is just arithmetic, base + size × (i - first).

Operation Time
read / write A[i] O(1)
add / remove at the end O(1)
add / remove at front or middle O(n) (everything after must shift)

Dynamic arrays

A static array's size is fixed at creation. A dynamic (resizable) array hides a pointer to a bigger-than-needed block and tracks size + capacity. On pushBack, if full, it allocates a new array of double capacity, copies everything over, and swaps. (C++ vector, Java ArrayList, Python list.) 

class DynamicArray:
    def __init__(self):
        self.cap, self.size = 1, 0
        self.data = [None] * self.cap

    def push_back(self, x):           # amortized O(1)
        if self.size == self.cap:     # full -> grow
            self._resize(2 * self.cap)
        self.data[self.size] = x
        self.size += 1

    def _resize(self, new_cap):       # O(n) copy, rare
        new = [None] * new_cap
        for i in range(self.size):
            new[i] = self.data[i]
        self.data, self.cap = new, new_cap

struct DynamicArray {
    int* data; int size = 0, cap = 1;
    DynamicArray() { data = new int[cap]; }

    void push_back(int x) {           // amortized O(1)
        if (size == cap) resize(2 * cap);  // full -> grow
        data[size++] = x;
    }
    void resize(int new_cap) {         // O(n) copy, rare
        int* nw = new int[new_cap];
        for (int i = 0; i < size; ++i) nw[i] = data[i];
        delete[] data; data = nw; cap = new_cap;
    }
};

Amortized analysis (aggregate method)

A single pushBack is O(n) in the worst case (the copy) — but those are rare. Doubling means the total copy work across n pushes is 1 + 2 + 4 + … + n ≈ 2n.

n pushBacks cost
total O(n)
per push (amortized) O(1)

"Amortized O(1)" = averaged over a long run of operations, not worst-case-per-call. Doubling wastes at most half the space; growing by 1.5× wastes less, by 3× wastes more.

Language notes

  • list is a dynamic array of object pointers; append over-allocates geometrically → amortized O(1).
  • insert(0, x) and pop(0) are O(n) (everything shifts) — use collections.deque for O(1) at both ends.
  • Storing pointers means an extra indirection per element; array.array / numpy pack raw values contiguously for cache-friendliness.
  • std::vector<T> stores T contiguously and grows ~2×; push_back is amortized O(1).
  • reserve(n) pre-allocates to skip reallocations; a reallocation invalidates all iterators/pointers into the vector.
  • insert/erase in the middle is O(n); shrink_to_fit releases the spare capacity.
