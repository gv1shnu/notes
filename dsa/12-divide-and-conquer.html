Divide and Conquer

Difficulty: Medium

A design paradigm, not one algorithm. Break a problem into smaller subproblems of the same kind, solve each recursively, then combine the results.

Move What
Divide split into subproblems (ideally each a fraction of n)
Conquer solve the subproblems recursively
Combine stitch their answers into the full answer

You've already met the two classics under their own topics: binary search (one half-size subproblem → O(log n)) and merge sort / quicksort (two half-size subproblems → O(n log n)). This page is about how to analyze any such algorithm.

Watch it run

Merge sort as a recursion tree. Divide walks top-down, halving the array until each piece is a single element (trivially sorted); then combine merges sorted halves back up the tree (green), sorted slice by sorted slice, until the whole array is ordered at the root. Reset shuffles.

Recurrence relations

The cost of a D&C algorithm is written as a recurrence: the work of the subproblems plus the divide/combine work.

Algorithm Recurrence Solves to
Binary search T(n) = T(n/2) + O(1) O(log n)
Merge sort T(n) = 2T(n/2) + O(n) O(n log n)

A faster algorithm hides in the split: Karatsuba

Multiplying two n-digit numbers the schoolbook way is O(n²). Naive D&C splits each number in half and does 4 half-size multiplications — still O(n²). Karatsuba's trick rewrites it to use only 3:

Subproblems Recurrence Time
4 T(n) = 4T(n/2) + O(n) O(n²)
3 T(n) = 3T(n/2) + O(n) O(nlog²3) ≈ O(n1.58)

One fewer recursive call changes the whole growth rate — the heart of clever D&C. 

def karatsuba(x, y):               # x, y nonnegative ints
    if x < 10 or y < 10:           # base case
        return x * y
    half = max(len(str(x)), len(str(y))) // 2
    hi_x, lo_x = divmod(x, 10**half)   # split each number in two
    hi_y, lo_y = divmod(y, 10**half)
    a = karatsuba(hi_x, hi_y)          # 3 recursive mults, not 4
    b = karatsuba(lo_x, lo_y)
    c = karatsuba(hi_x + lo_x, hi_y + lo_y)
    return a * 10**(2*half) + (c - a - b) * 10**half + b

long long karatsuba(long long x, long long y) {  // small operands only
    if (x < 10 || y < 10) return x * y;          // base case
    int half = max(num_digits(x), num_digits(y)) / 2;
    long long p = pow10(half);                  // split point 10^half
    long long hi_x = x / p, lo_x = x % p;
    long long hi_y = y / p, lo_y = y % p;
    long long a = karatsuba(hi_x, hi_y);         // 3 recursive mults
    long long b = karatsuba(lo_x, lo_y);
    long long c = karatsuba(hi_x + lo_x, hi_y + lo_y);
    return a * pow10(2*half) + (c - a - b) * p + b;
}

The Master Theorem

A formula for T(n) = a·T(n/b) + O(nd) — no recursion tree needed. Compare d with logba:

Condition T(n) Who dominates
d > logba O(nd) the divide/combine work (top)
d = logba O(nd log n) balanced across levels
d < logba O(nlogba) the leaves (subproblems)
Algorithm a, b, d Result
Binary search 1, 2, 0 case 2 → O(log n)
Merge sort 2, 2, 1 case 2 → O(n log n)
Karatsuba 3, 2, 1 case 3 → O(nlog²3)

Language notes

  • int is arbitrary-precision, and CPython already switches to Karatsuba for big multiplications internally — so x * y is fast; you'd only write the above to learn it.
  • divmod(x, 10**k) is the idiomatic split into high/low halves.
  • Deep D&C recursion can hit the recursion limit; most of these (merge sort, Karatsuba) are O(log n) deep, so it's rarely an issue.
  • Native integers are fixed-width and overflow silently — real big-number work needs __int128 (up to 127 bits) or a big-int library (boost::multiprecision).
  • The snippet assumes helper num_digits/pow10; a production version stores digits in a vector and multiplies limb by limb.
  • Past large sizes, FFT-based multiplication (O(n log n)) overtakes Karatsuba's O(n1.58).
