Search engines and Google dorks

Difficulty: Easy

Search engines have already crawled the target for you. They reveal indexed pages, public profiles, exposed files, and naming patterns — all without touching the target once. Fetching page after page by hand is a waste of time; being specific is the entire skill.

Beyond Google there is Bing, Yahoo, Brave, Yandex, DuckDuckGo, and SearXNG — an open-source metasearch engine that aggregates results from several providers. Functional, not always optimal. Different engines index differently, so a quiet result on one can be loud on another.

Google dorks

A "dork" is a targeted query built from search operators. Regular search says "here are results." Dorking says "here are results that forgot they were results."

Operator Finds
site:Pages on one domain only
filetype: / ext:A specific file type (pdf, xlsx, sql, env)
intitle: / inurl:A word in the page title or URL
intext:A word in the page body
cache:Google's cached copy of a page
before: / after:Date-bounded results (YYYY-MM-DD)
-wordExclude a term

Combine them. To hunt for resume PDFs left on a specific site: 

site:newtonschool.co filetype:pdf resume

The results are indexed PDF resumes hosted on that domain — names, emails, and phone numbers the site never meant to expose to a search operator. Or, to find resumes of a company's employees anywhere on the web: 

intitle:resume "software engineer" "at Google"

A few more that pay off on real targets: 

site:target.com intitle:"index of"          # open directory listings
site:target.com ext:sql | ext:env | ext:log  # leaked configs and dumps
site:target.com inurl:admin | inurl:login     # exposed panels

The Google Hacking Database is a curated list of these patterns. When you practise, run them against an in-scope lab target, not random companies — the query is passive, but the follow-up rarely stays that way.
