Mobile

Difficulty: Medium

A mobile app is two targets in one: the app on the device, and the backend API it talks to. The backend is often the bigger prize and is tested like any web API. The app itself is worth taking apart for what developers leave inside it.

What to check

  • Insecure local storage — data cached in cleartext on the device
  • Hardcoded secrets in the app package (API keys, tokens)
  • Weak API authentication behind the app
  • TLS problems, including whether certificate pinning can be bypassed
  • Insecure deep links and weak session handling
  • Assumptions that the device is not rooted or jailbroken

Cracking one open

An Android APK is a zip file. Unpack it and search the strings and resources for secrets before doing anything dynamic: 

$ unzip -q app.apk -d app/
$ grep -rniE 'api[_-]?key|secret|password|bearer' app/
app/res/values/strings.xml:  <string name="api_key">AIzaSyD...</string>

A hardcoded API key in strings.xml is a finding on its own, and often the key to the backend. Tools like jadx decompile the code for reading, and an intercepting proxy plus an emulator lets you watch (and test) the API calls — the OWASP Mobile Application Security guide is the checklist to work through.
