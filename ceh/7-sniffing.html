Sniffing

Sniffing is capturing traffic to understand protocols or prove sensitive data exposure. It usually assumes you are already on the network — a foothold, or a local LAN position — so it lives in the "assume breach" side of a black-box test.

Plaintext vs encrypted

The whole point lands in one comparison. An old plaintext protocol hands you the credentials directly: 

$ tshark -r capture.pcap -Y 'ftp.request.command == "PASS"'
FTP  USER labuser
FTP  PASS Summer2024!

Over TLS, the same login is just noise — you see the handshake and the certificate, but the payload is encrypted: 

$ tshark -r capture.pcap -Y 'tls.handshake.type == 11'
TLSv1.3  Certificate  CN=shop.lab.example.com
(application data: encrypted)

What makes a finding

  • Plaintext credentials on the wire (FTP, Telnet, HTTP Basic)
  • Weak or downgraded TLS
  • Insecure legacy protocols still in use
  • Missing network segmentation — traffic you should not be able to see
  • Noisy broadcast protocols leaking hostnames and users

In Wireshark, Statistics → Protocol Hierarchy is a fast way to see what is talking on a segment, and "Follow Stream" reconstructs a whole conversation. Capture is passive; the finding is "this data crosses the network unprotected," proven with a redacted snippet.
