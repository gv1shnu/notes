DNS and subdomains

A subdomain is a domain that is part of a larger one. If the main domain is example.com, then mail.example.com and blog.example.com are subdomains. Each one may point at a different server, and each server is a fresh piece of attack surface. Finding them, and the IPs behind them, reveals hosting providers, CDNs, exposed services, and the technology in use.

nslookup — name to IP

The simplest lookup turns a hostname into an address: 

$ nslookup shop.lab.example.com
Server:   1.1.1.1
Address:  1.1.1.1#53

Non-authoritative answer:
Name:     shop.lab.example.com
Address:  10.10.10.5

One host, one IP. Do this for every name you find and the network starts to take shape. (dig does the same job with more detail — covered in attribution.)

crt.sh — certificate transparency

Every time a site gets a TLS certificate, the certificate is published to public logs. Those logs are searchable, and they leak subdomains for free — including internal-sounding ones the site never linked anywhere: 

$ curl -s 'https://crt.sh/?q=%25.lab.example.com&output=json' | jq -r '.[].name_value' | sort -u
admin.lab.example.com
api.lab.example.com
dev.lab.example.com
shop.lab.example.com
staging.lab.example.com
vpn.lab.example.com

That staging. and dev. pair is exactly the kind of thing worth scanning later — test environments are often weaker than production. This is passive: you queried a public log, not the target.

DNSDumpster and Netcraft

Two free web tools do a lot of passive DNS work in one screen.

DNSDumpster maps a domain's hosts: A and MX records, name servers, IP addresses for each subdomain, server locations, and a visual graph of how they relate. From one lookup you often walk away with mail servers, hosting regions, and a shortlist of live hosts.

Netcraft's site report adds background: who administers the DNS, whether a CDN like Cloudflare is in front, the server operating system, and the technology stack. A typical report tells you something like "DNS on AWS, fronted by Cloudflare, Node.js hints" — before you have sent the target a single packet.

Source What you walk away with
nslookup / digIP for one hostname
crt.shSubdomain list from TLS certificate logs
DNSDumpsterA/MX/NS records, subdomain IPs, relationship map
NetcraftHosting, CDN, OS, and tech-stack hints

Cross-reference them. A subdomain that shows up in crt.sh but not DNSDumpster is worth a manual check — the gaps between sources are often where the interesting hosts hide.
