IoT and OT

Difficulty: Medium

IoT (connected devices) and OT (operational technology — industrial control systems) share one overriding constraint: the devices can be fragile or safety-impacting. A scan that would be routine against a web server can knock a PLC offline, and in an OT environment that can have physical consequences. Caution here is not politeness; it is safety.

What to check

  • Default credentials — the classic sticker-on-the-device password
  • Outdated firmware with known vulnerabilities
  • Exposed management interfaces reachable from the network
  • Weak or missing network segmentation between IT and OT
  • Insecure protocols (many industrial protocols have no authentication at all)
  • Vendor update gaps — devices that are never patched

If it has an app, a cloud account, and a password printed on a sticker, it has a personality and probably a CVE.

How to test it — carefully

Without dedicated hardware or a test rig, IoT and especially OT work is often literature and configuration review rather than active probing: check firmware versions against advisories, review network segmentation on paper, and confirm which protocols are in use. When active testing is in scope, it happens on a lab replica or in a maintenance window with the owners watching — never improvised against live equipment. MITRE ATT&CK for ICS is the reference for what matters in these environments.
