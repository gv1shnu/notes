Web content discovery

Difficulty: Medium

The visible site is the lobby. The job is the rest of the building: hidden directories, forgotten admin panels, a backup.zip someone left in the web root, and — most valuable in modern apps — the API surface behind the JavaScript. Two complementary moves get you there.

Brute-force the paths

A wordlist tool requests thousands of common paths and reports which exist. ffuf and gobuster are the usual choices: 

$ ffuf -u https://shop.lab.example.com/FUZZ -w common.txt -mc 200,301,401,403 -rate 50

admin        [Status: 401, Size: 21]     # exists, needs auth — interesting
backup       [Status: 200, Size: 90210]  # a directory listing? open it
robots.txt   [Status: 200, Size: 142]
api          [Status: 200, Size: 512]

This is loud, high-volume traffic against one host, so pace it (-rate) and watch for the target pushing back with 429s or a sudden wall of 403s (the WAF waking up). Always read robots.txt and sitemap.xml first — sites helpfully list the paths they want hidden.

Read the app's own JavaScript

Single-page apps hand you a gift: their frontend bundles literally contain the API endpoints they call. Fetching and grepping that JavaScript is nearly passive and often richer than any wordlist: 

# pull endpoints out of a bundled app.js
$ curl -s https://shop.lab.example.com/static/app.js | grep -oE '"/(api|v[0-9]|graphql)[^"]*"'
"/api/v1/orders"
"/api/v1/user/profile"
"/api/v1/admin/users"
"/graphql"

/api/v1/admin/users was never linked in the UI, but the JavaScript admitted it exists. Also probe for swagger.json, openapi.json, and a GraphQL introspection query — any one of them can hand you the entire API map at once.

The handoff: content discovery produces a list of reachable endpoints — login, search, cart, /api/v1/orders?id=…, an admin panel. That list is where the web-attacks chapter begins.

Decision tree — find the hidden surface

Spend the free, nearly-passive wins before you make noise, then read the status codes the brute force hands back: 

Target web app
│
├─ FREE wins first (nearly passive)
│     ├─ robots.txt / sitemap.xml → paths they tried to hide
│     ├─ grep the app.js bundle for "/api/…" → hidden endpoints
│     └─ swagger.json / openapi.json / GraphQL introspection
│
└─ Then brute-force paths (loud)  →  ffuf / gobuster -w wordlist
      │
      ├─ 200        → exists → open it
      ├─ 401 / 403  → exists but gated → interesting
      ├─ 301 / 302  → follow the redirect
      └─ 429 / wall of 403 → WAF waking up → slow down (-rate)
