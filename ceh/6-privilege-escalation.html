Privilege escalation

A foothold is usually a low-privilege account. Privilege escalation is showing whether that foothold can become root, SYSTEM, or Domain Admin. The method is always the same: enumerate the machine thoroughly, spot one misconfiguration, and use it. The automated helpers LinPEAS and WinPEAS run these checks for you — but know what they are looking for.

Linux

A handful of commands surface most local privesc paths: 

$ id                                   # what am I?
$ sudo -l                              # what can I run as root?
User labuser may run (root) NOPASSWD: /usr/bin/find

$ find / -perm -4000 2>/dev/null       # SUID binaries (run as owner)
/usr/bin/pkexec
/usr/bin/find

That sudo -l result is a straight path to root: GTFOBins shows that find can spawn a shell, and it is allowed to run as root with no password. Common Linux vectors: weak sudo rules, SUID binaries, writable cron jobs, exposed secrets, world-writable service files, and kernel exploits (last resort — they crash boxes).

Windows

The Windows equivalent starts with your privileges and the patch level: 

C:\> whoami /priv
SeImpersonatePrivilege        Enabled

C:\> systeminfo | findstr /B "OS Version"

SeImpersonatePrivilege is a classic win — the "Potato" family of tools (e.g. PrintSpoofer) turns it into SYSTEM. Other common vectors: unquoted service paths, weak service permissions (a service you can replace), AlwaysInstallElevated, and stored credentials.

Look for Because
sudo NOPASSWD / SUID (Linux)Runs your code as root
Writable cron or service fileYour payload runs on a schedule, as root/SYSTEM
SeImpersonate (Windows)Token impersonation to SYSTEM
Unquoted service pathWindows may execute your binary instead of the service
Secrets in config / historyReused passwords open the next door

Prove it by reading a protected file or showing the elevated shell — then stop at the agreed boundary. You do not need to dump /etc/shadow to prove you reached root.
