Attribution

Difficulty: Medium

In a black-box test, the first real mistake is assuming a URL points where you think it does. shop.lab.example.com might resolve to the client's own server, a shared CDN edge, a SaaS tenant they do not own, or a cloud IP that was someone else's last week. Touching the wrong box is not a technicality — it is the difference between a pentest and an intrusion. Attribution is proving a target is really in scope before any packet with intent leaves your machine.

"It was in the same subnet" has never once worked as a legal defence. The subnet does not love you.

Resolve every IP, then find the owner

Start by resolving the host — all of it, because geo-DNS and round-robin love to hide addresses behind the first answer: 

$ dig +short shop.lab.example.com
104.21.5.12
172.67.20.9

$ whois 104.21.5.12 | grep -iE 'orgname|netname'
OrgName:  Cloudflare, Inc.
NetName:  CLOUDFLARENET

That answer changes everything: you are talking to Cloudflare's edge, not the origin server. Your later scans will describe the CDN, not the app behind it. Map each IP to its ASN and owner and label it — client, CDN, or cloud provider.

Owner reads as… Meaning
The client's own org / ASNLikely a real origin host. In scope if listed.
A CDN (Cloudflare, Akamai, Fastly)You see the edge. Origin discovery is a separate job.
A cloud provider (AWS, Azure, GCP)Confirm the tenant is the client's; provider rules apply.

The rule: deny by default

Cross-check every resolved IP against the engagement's allowlist and forbidden list. If any address is outside scope, in the forbidden list, or on shared infrastructure you cannot attribute, stop. And do it again right before any active phase, not just once at the start — DNS drifts, and an in-scope name can quietly start resolving to out-of-scope infrastructure between one day and the next.

Surface mapping (passive first)

With attribution settled, the map you built in this chapter — crt.sh subdomains, DNSDumpster hosts, archive.org's old URLs — is your passive surface. Its size is the ceiling on everything later: an endpoint you never find is a bug you never find. Only once you have squeezed the passive sources do you resolve those candidates and send them a single, polite request to see which are alive. That step crosses from passive into active, so it runs under scope control, capped, and stopped if a firewall starts answering for everyone.

(In the campaign tracker, a passive collector writes this inventory to recon/inventory.json and a human reviews it before scanning. The review is the point; the file is just where it is written down.)
