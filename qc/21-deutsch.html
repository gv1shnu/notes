Deutsch

The granddaddy — historically the earliest quantum algorithm, though we met its friendlier cousin Bernstein–Vazirani first. Same toy-sized problem, same clean win over classical. You get a one-bit oracle f and a promise: it's either constant or balanced. Which one?

Type f(0), f(1) Coin analogy
Constant same (0,0 or 1,1) heads–heads / tails–tails
Balanced different (0,1 or 1,0) heads–tails / tails–heads

One query vs two

Classically you have no choice: evaluate f(0), evaluate f(1), compare. Two queries — because you must see both outputs to know if they match. The Deutsch algorithm answers in one, using the now-familiar three-beat:

Step Action
1 H the input qubit; ancilla in |−⟩
2 apply the oracle (phase kickback)
3 H the input qubit, then measure
Read 0 → constant, 1 → balanced

The single query is asked in superposition, so the oracle stamps both f(0) and f(1) into phases at once; interference then compares them without you ever reading either value. You never learn f(0) or f(1) individually — only the one global fact you actually asked for.

It scales: the Deutsch–Jozsa version takes an n-bit function promised constant or balanced. Classically that's up to 2n−1+1 queries; quantumly, still one. The toy grows teeth.
