Quantum Oracles

Every quantum algorithm ahead — Bernstein–Vazirani, Deutsch, Grover — is built around an oracle (a "black box"): a function f you can query but not look inside. Feed it an input, get an output, no peeking at the gears. The interesting question becomes "how few queries do I need to learn something about f?"

Making a function reversible

Gates are reversible, but a function like f(x) usually isn't. The fix is to never overwrite the input — XOR the answer into a separate output qubit:

Uf : |x⟩|y⟩ → |x⟩|y ⊕ f(x)⟩

Apply Uf twice and you're back where you started (y ⊕ f ⊕ f = y), so it's unitary. The input |x⟩ survives untouched — which is the whole point.

The trick that makes it useful: phase kickback

Run the oracle on a plain output qubit and nothing exciting happens. But prep that qubit in |−⟩ first (X then H), and something sneaky occurs:

f(x) Effect on |x⟩
0 nothing
1 multiply |x⟩ by −1 (a phase flip)

The function's answer doesn't flip the output qubit — it gets kicked back into the phase of the input. And phase, sandwiched between Hadamards, becomes a measurable result through interference. Combine that with Hadamards putting every input into superposition, and a single oracle call touches all inputs at once, writing each answer into a phase.

Why it matters: Hadamard everything → one oracle query (answers land in phases) → Hadamard again (interference reads them out). That three-beat is the skeleton of every algorithm in the next few pages; only the oracle changes.
