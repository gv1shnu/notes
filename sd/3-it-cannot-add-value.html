The disease: functional decomposition

3. It cannot add value.

The first law of thermodynamics, in layman's terms: you cannot add value without sweating. Architecture is high-value work. Hand someone a spec that says "do A, do B, do C" and they say "no sweat — A block, B block, C block." They just tried to add value without sweating. The universe forbids it.

House by function What it costs
build the kitchen first foundation, walls, plumbing, roof — "done"
now add a bedroom jackhammer the foundation, lift the kitchen, rebuild it next to the bedroom

You'd fire any builder who worked this way — yet it's how software gets built. And naming the rooms doesn't save you: domain decomposition (kitchen = cooking, bedroom = sleeping) is just functional decomposition in disguise. A rose by any other name.
