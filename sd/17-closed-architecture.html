Validating the design

Difficulty: Hard

17. Closed architecture.

How may layers call each other?

Direction Verdict
Up worst — imports high-level volatility into low layers
Sideways bad — a manager calling a manager? then it's really an engine
Down, exactly one layer the rule — maximizes encapsulation

That's a closed architecture: everyone may call exactly one layer below, no more. Open (call anyone) is flexible but couples everything; favour closed.

Sanctioned escape hatches:

Allowed Note
Utilities, used by all layers the cappuccino test: if you can't explain using it in a coffee machine, it isn't a utility
Manager → manager, queued only technically a call down (queue = resource); for deferred "do this later" work
Managers & engines both call ResourceAccess engines are a strategy of the manager — same layer

When a developer wants to "open up" the design, find the real need. Manager wants to call a client? There's volatility in who's notified — route it through a Pub/Sub utility instead.
