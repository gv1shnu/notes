The disease: functional decomposition

4. The testing trap.

A jumbo jet where every component passes unit testing flawlessly — would you board it? No. Defects live in the interactions between components, not inside any one of them. The fuel pump is fine, the pipe is fine, but a g-force turn makes a cavitation bubble that stalls the engine.

Testing Catches
Unit defects inside one component
Regression defects in the interaction between subsystems (the real ones)

A functionally-decomposed system is so coupled you can't delineate it for regression testing — so it's effectively untestable, so it's rife with defects. Developers fall back on unit testing not because it's valuable but because it's all they can do.

That's searching under the streetlight: a man looks for his keys under the lamp, not where he dropped them, "because the light is better here."
