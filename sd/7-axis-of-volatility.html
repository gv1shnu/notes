The cure: volatility-based decomposition

7. The axis of volatility.

There are only two ways anything can change. Ask both about every piece of the system:

Axis Question
Same customer, over time will one customer want this differently in the future?
All customers, right now do different customers already use this differently?

Example — a house:

Over time Across customers
furniture, appliances, occupancy, utilities structure, neighbours, city / building code

The axes are independent — what changes on one is rarely what changes on the other. If something changes on both axes at once, that's a warning sign it's actually functional decomposition sneaking back in.
